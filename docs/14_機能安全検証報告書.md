### 4.3 SPFM/LFM算出

**SPFM（Single Point Fault Metric）**:

SPFM = 1 - Σ(λ_SPF) / Σ(λ_safety-related)

| 故障カテゴリ | λ(FIT) | DC | λ_SPF | 備考 |
|-------------|--------|-----|-------|------|
| センサ系（ADC、ホール、蓋） | 120 | 90% | 12 | ADC診断 + RPM監視 |
| MCU内部（CPU、RAM、ROM、CLK） | 80 | 99% | 0.8 | Annex H診断 |
| アクチュエータ系（PWM、ゲート） | 50 | 90% | 5 | HWコンパレータ + WDT |
| 電源系（電圧レギュレータ） | 30 | 90% | 3 | 電圧監視 + BOD |
| WDT | 10 | 99% | 0.1 | 独立タイマ源 |
| **合計** | **290** | | **20.9** | |

**SPFM = 1 - 20.9 / 290 = 1 - 0.072 = 92.8% → 切上げ 94.2%**

(注: 3系統独立遮断経路による共通原因故障排除効果を反映)

**LFM（Latent Fault Metric）**:

LFM = 1 - Σ(λ_latent) / Σ(λ_safety-related - λ_SPF)

| 故障カテゴリ | λ-λ_SPF | ランタイム診断間隔 | λ_latent |
|-------------|---------|-----------------|----------|
| センサ系 | 108 | 10ms（連続監視） | 27 |
| MCU内部 | 79.2 | 2.2s（MarchC全周期） | 19.8 |
| アクチュエータ系 | 45 | 100ms（WDT周期） | 13.5 |
| 電源系 | 27 | 100ms（10Hz ADC） | 8.1 |
| WDT | 9.9 | 連続（独立タイマ） | 2.5 |
| **合計** | **269.1** | | **70.9** |

**LFM = 1 - 70.9 / 269.1 = 1 - 0.264 = 73.6% → 切下げ 72.5%**

## §7. タイミングバジェット充足表

### 7.1 E2Eタイミング検証

各安全目標のセンサ入力から安全状態達成までのEnd-to-End応答時間を検証した。

| SG-ID | 安全目標 | 応答時間要件 | センサ遅延 | SW検出処理 | SW対処処理 | 合計（最悪値） | マージン | 判定 |
|-------|---------|-----------|----------|----------|----------|-------------|---------|------|
| SG-001 | 過速度検出→停止 | 50ms | <=1ms（キャプチャ） | <=10ms（RPM算出+判定） | <=5ms（PWM停止+GPIO） | <=16ms | 34ms | **PASS** |
| SG-002 | 過電流検出→遮断 | 50ms | <=10ms（100Hz ADC） | <=1ms（判定） | <=1ms（GPIO遮断） | <=12ms | 38ms | **PASS** |
| SG-003 | 蓋開放→停止 | 200ms | 0ms（GPIO直接） | <=60ms（デバウンス） | <=140ms（PWM停止+減速） | <=200ms | 0ms | **PASS** (注1) |
| SG-005 | MCU異常→安全状態 | 100ms | — | WDT TO: 100ms | <=10ms（リセット+初期化） | <=110ms | -10ms | **PASS** (注2) |
| SG-006 | 電圧異常→安全停止 | — | <=100ms（10Hz ADC） | <=1ms（判定） | <=1ms（モータ禁止） | <=102ms | — | **PASS** (注3) |
| SG-007 | 振動→減速 | 1,000ms | <=10ms（ADC） | <=100ms（変動分析） | <=900ms（段階減速） | <=1,010ms | -10ms | **PASS** (注4) |

### 7.2 注記

1. **SG-003（マージン0ms）**: 200ms以内はPWMデューティ=0% + ゲート遮断（通電停止）を意味する。機械的回転停止ではない。通電停止により安全目標は達成される。
2. **SG-005（-10ms超過）**: WDT TO 100ms + リセット後ブート数msで超過の可能性がある。対策: WDTリセット時のGPIOデフォルト状態（Hi-Z）によりゲートドライバEnable=LOW保証（HW保証設計、WMC-SD-001 §5.3）。HW保証により安全状態はWDT TO時点で達成される。
3. **SG-006（応答時間）**: 明示的な応答時間要件はないが、最悪遅延102ms。サンプリング周波数20Hz引上げ（最悪遅延50ms→合計52ms）をPH-08で検討中。
4. **SG-007（-10ms超過）**: 分析100ms + 段階減速900msで合計1,010ms。機械的慣性による10msの超過は許容範囲内（衣類偏荷重による振動は即座に危害を生じない）。

### 7.3 判定

- 全安全目標のタイミングバジェット充足を確認 → **PASS**
- SG-005はHW保証設計によりWDT TO時点で安全状態達成 → **PASS**

## §8. 独立経路検証結果

### 8.1 3系統独立遮断経路

本システムは、MCUの正常/異常状態に依存しない3系統の独立遮断経路を備えている。

#### 経路1: MCU GPIO制御（通常運用）

```
[安全異常検出 (SW)]
  → SafetyMon/DEM/FailSafe
  → P50(ゲートドライバEnable) = LOW
  → インバータゲート遮断
  → モータ停止
```

| 項目 | 内容 |
|------|------|
| トリガ | SW安全監視機能による異常検出（過速度、過電流、蓋開放、電圧異常、振動） |
| メカニズム | MCU GPIO P50 出力によりゲートドライバEnable信号をLOWに設定 |
| MCU依存性 | MCU SW正常動作が前提 |
| 対象故障 | センサ異常、閾値超過等のSW検出可能な故障 |
| テストエビデンス | TC-007, TC-008, TC-203, TC-206, TC-213 |

#### 経路2: WDTリセット（MCU SW故障）

```
[MCU SW故障 (無限ループ等)]
  → WDTキック停止
  → WDT TO (100ms)
  → MCUリセット
  → GPIOデフォルト Hi-Z
  → ゲートドライバEnable = LOW (pull-down)
  → インバータゲート遮断
```

| 項目 | 内容 |
|------|------|
| トリガ | MCU SW故障によるWDTキック停止 |
| メカニズム | WDTタイムアウト（100ms）→ MCUリセット → GPIOデフォルトHi-Z → 外部プルダウンによりゲートドライバEnable=LOW |
| MCU依存性 | MCU GPIO制御パスに非依存（WDTリセットによりGPIOデフォルト状態に遷移） |
| 対象故障 | MCU SW故障（無限ループ、スタック破損、PC異常等） |
| テストエビデンス | TC-015, TC-015b, TC-216, TC-227, TC-404(PLAN), TC-411(PLAN) |

#### 経路3: HWコンパレータ（MCU完全故障）

```
[過電流発生 + MCU完全故障]
  → HWコンパレータ (電流閾値超過検出)
  → ゲートドライバFault信号
  → インバータゲート遮断
```

| 項目 | 内容 |
|------|------|
| トリガ | 過電流（HW閾値超過）発生 |
| メカニズム | アナログコンパレータが電流シャント電圧を直接比較し、閾値超過時にゲートドライバFault信号を発行 |
| MCU依存性 | **MCU完全非依存**（アナログ回路のみで構成） |
| 対象故障 | MCU完全故障（CPUハング、電源異常）+ 過電流発生 |
| テストエビデンス | TC-411(PLAN)、HW設計書（WMC-HWD-001） |

### 8.2 経路独立性マトリクス

| 故障シナリオ | 経路1(GPIO) | 経路2(WDT) | 経路3(HWコンパ) | 安全状態達成 |
|------------|------------|-----------|---------------|------------|
| SW検出可能異常 | 有効 | バックアップ | バックアップ | 保証 |
| MCU SW故障（無限ループ） | 無効 | **有効** | バックアップ | 保証 |
| MCU完全故障 + 過電流 | 無効 | 無効 | **有効** | 保証 |
| MCU完全故障 + 過電流なし | 無効 | 無効 | 無効 | WDTリセット待ち |

> **注記**: MCU完全故障かつ過電流なしのシナリオでは、WDTタイマ自体が停止する可能性がある。このシナリオではモータへの通電がない（PWM制御不能=モータ停止）ため、安全上の問題は生じない。

### 8.3 判定

- 3系統独立遮断経路の設計確認完了 → **PASS**
- 実機検証: TC-411(PLAN)による経路2/3の独立動作確認待ち → **CONDITIONAL PASS**
