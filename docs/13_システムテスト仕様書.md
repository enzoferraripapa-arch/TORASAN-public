## 1. テスト方針

### 1.1 テスト目的

PH-03 HARAで定義された安全目標（SG-001〜SG-006）およびPH-04 FSCで定義された機能安全要求（FSR-001〜FSR-010）に対し、システムレベルの妥当性確認テストを実施する。実機モータ負荷装置を用いたエンドツーエンド検証により、安全目標が達成されることを確認する。

### 1.2 テスト戦略

- **テスト手法**: システムレベル妥当性確認（実機テスト）
- **カバレッジ目標**: SG 6件全件カバー（SG-004はHW独立保護のため除外記載）、FSR 10件全件カバー
- **テスト分類**: 安全機能検証 / 通常動作検証 / エンドツーエンドシナリオ
- **判定基準**: HARA記載の安全目標受入基準（応答時間・閾値）に基づく判定
- **テスト方針**: ブラックボックス（外部入出力のみで判定）

### 1.3 テスト環境

| 項目 | 内容 |
|------|------|
| ターゲット | Renesas RL78/G14 (R5F104BG) 評価ボード |
| モータ負荷装置 | BLDCモータ (1500W定格) + 可変負荷装置 |
| 電流計測 | クランプ電流計（AC/DC、応答10ms以下） |
| 電圧可変電源 | プログラマブル電源（4.0〜6.0V可変） |
| オシロスコープ | 4chデジタルオシロ（応答時間計測） |
| 蓋模擬装置 | マイクロスイッチ + ソレノイド（手動操作可能） |
| データロガー | 全センサ値リアルタイム記録 |
| ストップウォッチ | 高精度タイマ（応答時間計測補助） |

### 2.2 SG-004 除外理由

SG-004（漏電遮断）はHW独立保護機構（RCD 30mA/30ms）により実現される。SW制御対象外のため、本テスト仕様書ではSWテストケースを定義しない。RCDの動作確認はHW単体テスト（PH-12外の電気安全試験）で実施する。トレーサビリティ上はSG-004 = HW独立保護として記録する。

## 3. テストケース詳細

### 3.1 過速度検出→モータ停止（SG-001 / FSR-001, FSR-002）

#### TC-401: 過速度検出→モータ停止 エンドツーエンド検証（検出）

| 項目 | 内容 |
|------|------|
| **テストID** | TC-401 |
| **対象SG** | SG-001 |
| **対象FSR** | FSR-001（検出）, FSR-002（反応） |
| **カテゴリ** | 検出 |
| **優先度** | 高 |

**前提条件**: システム起動完了、起動時診断PASS、OP-03（スピン）運転中、1200rpm安定

**入力**:
- モータ負荷を急減させて回転数を1500rpm超へ上昇させる

**テスト手順**:
1. OP-03スピンモードで1200rpm安定運転を確認
2. 負荷装置の制動トルクを急減させ、回転数を上昇させる
3. 1500rpm超過時点をオシロスコープでトリガ
4. モータ停止（PWM=0%、EN=LOW）までの応答時間を計測
5. 停止後の状態（OP-05）を確認

**期待結果**:
- 1500rpm超過を検出
- 100ms以内にモータ停止（PWM全相OFF、EN=LOW）
- 状態遷移: OP-03 → OP-05（異常停止）
- 蓋ロック維持

**PASS/FAIL判定基準**:
- 過速度検出から停止完了まで ≦ 100ms
- FTTI 500ms以内に安全状態到達

**テスト結果**: 実施待ち

**安全分類**: Class B
**トレーサビリティ**: SG-001 → FSR-001/002 → TSR-001/002/003
