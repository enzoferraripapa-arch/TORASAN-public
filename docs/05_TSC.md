## 2. TSR詳細定義

### TSR-001: ホールセンサ回転数計測機能

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-001（過速度検出） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 入力: ホールセンサ3相（IF-02: P20, IF-03: P21, IF-04: P22）— 5V/GNDデジタル信号
- 計測手法: GPIO入力キャプチャ機能で立上り/立下り周期を計測
- クロック: 32MHz (`product_spec.mcu.clock_mhz`) — 16ビットカウンタ
- 計測精度: ±5rpm以内（1500rpm時）
- 更新周期: ホールセンサ信号遷移毎（最短: 約6.7ms @1500rpm/3相）
- 無信号検出: 200ms以上遷移なし → 回転停止と判定

**product_spec整合**:
- MCU: RL78/G14 32MHz → カウンタ分解能 31.25ns → 十分な精度

---

### TSR-002: 回転数閾値比較・過速度判定処理

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-001（過速度検出） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 判定閾値: **1500rpm** (`product_spec.motor.overspeed_rpm`)
- 判定周期: **10ms**（`product_spec.adc.current_hz` = 100Hzと同期）
- 判定ロジック: TSR-001計測値 ≧ 1500rpm → 過速度フラグON
- 検出遅延: ≦1判定周期（10ms）
- 確定条件: 1回の閾値超過で即判定（高速応答優先）
- トリガ: FSR-002（TSR-003）を起動

---

### TSR-003: PWM出力即時停止制御

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-002（過速度時モータ停止） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW+HW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 停止動作: 全PWMチャネル出力 = 0%（TOM0レジスタ強制クリア）
- HW連動: インバータゲートドライバEN信号 = LOW（P10出力）
- 応答時間: **≦100ms**（FSR-002定義 / `product_spec.wdt.timeout_ms`）
  - 内訳: 検出10ms + 判定即時 + PWM停止1ms以内 = 十分なマージン
- 状態遷移: OP-05（異常停止）に遷移
- 復帰条件: 電源リセット（ソフトリセット不可）
- UI通知: エラーコードをUART（IF-09）送信

---

### TSR-004: ADC電流値取得・過電流判定処理

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-003（過電流検出） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 入力: 電流センサADC（IF-05, ANI0）
- ADC: **10bit** / Vref **5.0V** (`product_spec.adc.bits` / `vref_v`)
  - 分解能: 5.0V / 1024 ≒ **4.88mV/LSB**
- サンプリング: **100Hz**（`product_spec.adc.current_hz`）→ 10ms周期
- 過電流閾値: **8A (8000mA)** (`product_spec.motor.max_a` / `max_ma`)
- デジタルフィルタ: 移動平均 N=4（フィルタ遅延 ≦40ms）
- 判定: フィルタ後値 ≧ 8A → 過電流フラグON
- HWバックアップ: ヒューズによる物理遮断（独立経路）

---

### TSR-005: インバータゲート遮断制御

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-004（過電流時通電遮断） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW+HW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 遮断動作: PWM全相OFF + インバータゲートEN = LOW（P10）
- HW連動: ゲートドライバHWリセット（SW信号とOR論理で遮断）
- 応答時間: **≦50ms**（FSR-004定義）
  - 内訳: ADC取得10ms + フィルタ遅延40ms + 遮断1ms以内
- 状態遷移: OP-05 → 電源リセットまで保持
- ログ: 過電流イベント（発生時刻、電流値）を不揮発メモリに記録

---

### TSR-006: 蓋センサGPIO監視・デバウンス処理

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-005（蓋開放検出） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 入力: 蓋センサGPIO（IF-07, P43, 内部プルアップ）
- 信号論理: 蓋閉 = LOW / 蓋開 = HIGH
- 監視周期: **5ms**（FSR-005定義 — 制御周期以下）
- デバウンス: **5ms以上**継続して開状態を確認後に判定（FSR-005定義）
- センサ診断: 定期的な信号遷移確認（断線検出: 連続HIGH > 10秒 → 診断）

---

### TSR-007: 蓋開放時モータ停止シーケンス

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-006（蓋開放時モータ停止） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 応答時間: **≦200ms**（FSR-006定義 — 検出からモータ停止まで）
- 停止動作: PWM出力 = 0%（段階的減速ではなく即時停止）
- 蓋ロック: IF-08（P44）でロック状態を維持
- 状態遷移: OP-05 → 蓋が閉じられるまで停止継続
- 復帰条件: 蓋閉鎖確認後、OP-04（待機状態）に遷移
- UI通知: 蓋開放表示（IF-09 UART）

### TSR-008: CPUレジスタテスト

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-007（MCU自己診断） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧95%） |
| 規格参照 | IEC 60730 Annex H Table H.1 — H.2.16.5 |

**技術仕様**:
- テスト対象: CPU汎用レジスタ（RL78/G14: AX, BC, DE, HL, R0-R7）
- テストパターン: 0xAAAA / 0x5555 全ビット読み書き
- 実行タイミング: スタートアップ時（全レジスタ一括）
- 所要時間: ≦1ms（32MHz動作時）
- 異常時動作: リセット → 安全状態遷移（TSR-012）

---

### TSR-009: RAM自己テスト（March C分割実行）

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-007（MCU自己診断） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧95%） |
| 規格参照 | IEC 60730 Annex H Table H.2 — H.2.19.6 |

**技術仕様**:
- テスト対象: 内部SRAM **5632B (5.5KB)** (`product_spec.mcu.ram_b`)
- テスト手法: **March C** (`product_spec.ram_test.algorithm`)
- スタートアップ: 全SRAM一括テスト（電源投入時、変数初期化前）
- ランタイム: 分割実行
  - ブロックサイズ: **256B** (`product_spec.ram_test.block_b`)
  - ブロック数: **22** (`product_spec.ram_test.block_count`)
  - 実行周期: 100ms/ブロック
  - 全域スキャン: **2.2秒** (`product_spec.ram_test.full_cycle_s`)
- データ退避: テスト対象ブロックのデータを退避バッファに保存後テスト、完了後復元
- 異常時動作: リセット → 安全状態遷移（TSR-012）

### TSR-010: Flash ROM CRC32検査

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-007（MCU自己診断） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧99%） |
| 規格参照 | IEC 60730 Annex H Table H.3 — H.2.19.8 |

**技術仕様**:
- テスト対象: プログラムFlash ROM **64KB** (`product_spec.mcu.flash_kb`)
- テスト手法: **CRC32** (`product_spec.crc_algorithm`)
- 参照値: プログラム書込み時にイメージから事前計算し末尾セクタに格納
- スタートアップ: 全Flash一括CRC32計算・参照値比較
  - 所要時間: ≦500ms（32MHz動作時、64KB対象）
- ランタイム: 分割CRC32検証（4KB/周期、全域スキャン ≦16秒）
- 異常時動作: CRC不一致 → リセット → 安全状態遷移（TSR-012）

### TSR-011: クロック監視（独立クロック比較）

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-007（MCU自己診断） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW+HW |
| DC目標 | 中（≧60%） |
| 規格参照 | IEC 60730 Annex H Table H.4 — H.2.18.10 |

**技術仕様**:
- メインクロック: **32MHz** (`product_spec.mcu.clock_mhz`)
- サブクロック: **15kHz** (`product_spec.mcu.sub_clock_khz`)
- 監視手法: メインクロックでサブクロック周期をカウントし、期待値と比較
- 許容偏差: **±4%** (`product_spec.clock_tolerance_pct`)
- 監視周期: 100ms（サブクロック周期の十分な平均化）
- HW連動: RL78/G14内蔵クロック監視回路（異常検出時割込み生成）
- 異常時動作: クロック異常割込み → 安全状態遷移（TSR-012）

### TSR-012: 安全状態遷移制御

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-008（MCU異常時安全遷移） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 応答時間: **≦100ms** (`product_spec.wdt.timeout_ms`)
- トリガ条件:
  - 自己診断失敗（TSR-008〜011）
  - WDTタイムアウト（TSR-015）
  - スタックオーバーフロー検出
- 遷移動作:
  1. 全PWM出力 = 0%（即時）
  2. インバータゲートEN = LOW（P10 HW信号）
  3. 全出力ポート無効化
  4. エラーフラグ設定（不揮発メモリ記録）
  5. エラーコード送信（IF-09 UART）
  6. 赤色LED点滅（IF-10）
- 状態保持: OP-05（異常停止）— 電源リセットまで保持
- 復帰: 電源OFF→ON → スタートアップ診断PASS → OP-04（待機）

---

### TSR-013: ADC電圧監視・判定処理

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-009（電圧監視） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 入力: MCU VDD電圧（IF-06, ANI1 — 分圧回路経由）
- ADC: **10bit** / Vref **5.0V** (`product_spec.adc.bits` / `vref_v`)
- 監視周期: **100ms** (10Hz = `product_spec.adc.voltage_hz`)
- 正常範囲: **4.5V〜5.5V** (`product_spec.voltage_monitor.min_v` / `max_v`)
- デジタルフィルタ: 移動平均 N=4
- 判定: フィルタ後値 < 4.5V or > 5.5V → 電圧異常フラグON
- HWバックアップ: 電圧監視IC（リセット生成 — 独立経路）

---

### TSR-014: 電圧異常時モータ起動禁止制御

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-010（電圧異常時モータ禁止） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- 禁止動作: モータ起動禁止フラグ（motor_inhibit）= TRUE
  - PWM出力コマンドを全て遮断
  - 新規起動要求を拒否
- 状態保持: 電圧が正常範囲に復帰するまで禁止継続
- 復帰条件: 電圧正常範囲に200ms以上継続 → 禁止解除可能（手動リセット優先）
- ブラウンアウト対策: 電圧低下時はモータ停止優先（MCU動作維持より安全）
- UI通知: 電圧異常エラー表示（IF-09 UART）
- ログ: 電圧異常イベント（時刻、電圧値）を不揮発メモリに記録

---

### TSR-015: WDTタイムアウト設定・キック管理

| 項目 | 内容 |
|------|------|
| 親FSR | FSR-007/FSR-008（MCU自己診断/安全遷移） |
| 安全分類 | IEC 60730 Class B |
| HW/SW割当 | SW+HW |
| DC目標 | 高（≧80%） |

**技術仕様**:
- WDTハードウェア: RL78/G14内蔵ウォッチドッグタイマ
- タイムアウト: **100ms** (`product_spec.wdt.timeout_ms`)
- キック周期: **50ms** (`product_spec.wdt.kick_ms`)
- キック条件: メイン制御ループの全処理（入力取得→制御演算→出力更新→自己診断→エラーチェック）が正常完了した場合のみキック
- 異常時: WDTタイムアウト → MCU強制リセット → スタートアップ診断 → 安全状態遷移（TSR-012）
- ウィンドウWDT: 非対応（RL78/G14はウィンドウWDT非搭載のため通常WDTを使用）

## 4. Class B安全分類確認

IEC 60730 Class B要件に基づく分類確認:

| 要件 | Class B条件 | 本システムの対応 | 判定 |
|------|-----------|---------------|------|
| 自己診断 | CPU/RAM/ROM/クロック診断必須 | TSR-008〜011 — Annex H準拠 | OK |
| 診断カバレッジ | DC ≧ 60% | 全体加重平均 87% | OK |
| フォールトリアクション | 危険出力の制御必須 | TSR-012 — 全PWM停止+ゲート遮断 | OK |
| ウォッチドッグ | 独立タイマによる監視 | TSR-015 — HW WDT 100ms | OK |
| 冗長性 | Class Bでは必須ではないが推奨 | 多層防御（SW + HWバックアップ） | OK |

**信号フロー**:
1. ホールセンサ(GPIO) → TSR-001計測 → TSR-002判定 → TSR-003停止 → FSR-001/002達成
2. 電流センサ(ADC) → TSR-004判定 → TSR-005遮断 → FSR-003/004達成
3. 蓋センサ(GPIO) → TSR-006デバウンス → TSR-007停止 → FSR-005/006達成
4. スタートアップ: TSR-008〜011自己診断 → FSR-007達成
5. 異常検出 → TSR-012安全遷移 → FSR-008達成
6. 電圧(ADC) → TSR-013監視 → TSR-014禁止制御 → FSR-009/010達成
7. WDT → TSR-015キック監視 → FSR-007/008達成（最終防壁）
