## 4. 安全クラス割当

IEC 60730はASIL分解ではなく、クラス分類を採用しています。本システムでは以下の判断により全要件を**Class B**に分類:

| クラス | 特性 | 本システム適用理由 |
|--------|------|-------------------|
| **Class A** | 単純安全機能 (冗長性なし) | - |
| **Class B** | 診断カバレッジ要求あり (DC≧60%) | ✓ 適用 |
| **Class C** | DC≧90% + 冗長機能 | - (簡素設計) |
| **Class D** | DC=100% + 冗長自動制御 | - |

**本システムの安全機構分類:**

| 安全機能 | 単独/冗長 | DC目標 | Class | 根拠 |
|---------|---------|--------|-------|------|
| 過速度検出・停止 | 単独 | 高 (≧80%) | Class B | 診断: 閾値比較+WDT |
| 過電流検出・遮断 | 単独 | 高 (≧80%) | Class B | 診断: フィルタ+WDT |
| 蓋インターロック | 単独 | 高 (≧80%) | Class B | 診断: デバウンス+WDT |
| 自己診断 | 実装 | 高 (≧90%) | Class B | Annex H準拠 |
| WDT | 実装 | 高 | Class B | ハードウェア監視 |

## 5. 診断カバレッジ (DC) 目標値

IEC 60730 Annex H に準拠した自己診断機構:

### 5.1 CPU診断 (Annex H Table H.1)

| 対象 | 検査手法 | DC目標 | 実装方式 | TBD |
|------|---------|--------|---------|------|
| CPU汎用レジスタ | 全ビット読み書きテスト | 高 (95%) | スタートアップのみ | - |
| CPU制御レジスタ | 機能別テスト | 中 (60%) | ソフトウェアテスト | WDTタイムアウト値 |

**実装例:**
```
スタートアップテスト:
  FOR each register R0 to R15:
    R = 0xAAAA; Check R == 0xAAAA
    R = 0x5555; Check R == 0x5555
  Result: PASS → メイン処理へ
          FAIL → リセット + 安全状態遷移
```

### 5.2 RAM診断 (Annex H Table H.2)

| 対象 | 検査手法 | DC目標 | 実装方式 | 範囲 |
|------|---------|--------|---------|------|
| 内部SRAM | March C+ テスト | 高 (95%) | スタートアップ+ランタイム | 5.5KB (5632B, スタック除外) |

**実装例:**
```
スタートアップ: March C+ (全SRAM 5.5KB)
  実行時間: 約50ms (CPU 32MHz時)

ランタイム: March C+ (分割実行)
  分割サイズ: 256B/周期
  実行周期: 50ms
  全スキャン時間: 1.125秒
```

### 5.3 ROM診断 (Annex H Table H.3)

| 対象 | 検査手法 | DC目標 | 実装方式 | 頻度 |
|------|---------|--------|---------|------|
| プログラムFlash ROM | CRC32チェックサム | 高 (99%) | スタートアップのみ | 起動時 1回 |

**実装例:**
```
参照値: プログラム書き込み時にイメージから計算
計算対象: 0x00000 ~ 0x0FFFF (64KB)
チェック周期: 起動時のみ (1秒以内に完了)
```

### 5.4 クロック診断 (Annex H Table H.4)

| 対象 | 検査手法 | DC目標 | 実装方式 | 検出時間 |
|------|---------|--------|---------|---------|
| 内部 vs 外部クロック | クロスモニタ (±4%) | 中 (70%) | ハードウェア監視回路 | <100ms |

**実装例:**
```
メインクロック: 32MHz (RL78/G14 仕様)
LOCO: 15kHz (内蔵低速オシレータ)
許容偏差: ±4% (32MHz MAIN vs 15kHz LOCO)

監視: ハードウェア周波数検出回路
  - クロック異常 → リセット
  - 周波数逸脱 → WDTトリガ
```

### 5.5 I/O診断 (Annex H Table H.6)

| 対象 | 検査手法 | DC目標 | 実装方式 | 周期 |
|------|---------|--------|---------|------|
| GPIO出力 (PWM) | プラウジビリティチェック | 中 (60%) | ソフトウェアチェック | 毎制御周期 |
| GPIO入力 (センサ) | 信号有無確認 | 中 (60%) | 機能テストのみ | - |

**実装例:**
```
PWM出力プラウジビリティ:
  PWM出力値 (期待値) と 実際の蓋/電流検出の関連を確認
  例: PWM=0 時に電流≠0 → エラー通知

GPIO入力:
  ホールセンサ信号: 周期信号確認 (制御周期内に1以上の遷移)
  蓋スイッチ: デバウンス後の状態確認
```

## 6. 安全機構の技術的実現方針

### 6.1 過速度保護メカニズム (FSR-001→002)

**概要:**
```
ホールセンサ信号 → GPIO入力キャプチャ → 周期計測
  → 回転数算出 → 閾値比較 → 過速度判定
  → PWM停止 → モータ停止
```

**技術詳細:**
- **ホールセンサ入力:** GPIO キャプチャ機能で立ち上がり/立ち下がり周期を計測
- **計測精度:** ±5 rpm以内 (12MHz クロック, 16ビット カウンタ)
- **判定ロジック:** 計測周期ごとに即座に比較実施
- **応答時間:** 最大 1制御周期 (10ms)

### 6.2 過電流保護メカニズム (FSR-003→004)

**概要:**
```
電流センサ出力 (シャント抵抗 Rshunt=1mΩ)
  → ADC変換 → デジタルフィルタ (移動平均)
  → 閾値比較 → 過電流判定 → ゲート遮断
```

**技術詳細:**
- **センサ:** シャント抵抗 1mΩ + 計測用オペアンプ
- **ADC:** 12ビット 1000サンプル/秒
- **フィルタ:** 移動平均 N=5 (フィルタ遅延 約5ms)
- **閾値:** 100A (コネクタ規格制限値) で遮断
- **応答時間:** 最大 10ms (フィルタ + 判定 + ゲート駆動)

### 6.3 蓋インターロック機構 (FSR-005→006)

**概要:**
```
蓋マイクロスイッチ → GPIO入力 → デバウンス
  → 状態判定 → 開閉検出 → 停止シーケンス
```

**技術詳細:**
- **センサ:** 蓋マイクロスイッチ (LOW=閉, HIGH=開)
- **デバウンス:** ヒステリシス型 20ms (チャタリング除去)
- **状態遷移:**
  - 開く検出 → 緊急停止シーケンス開始
  - 閉じる検出 → 停止フラグクリア後、主制御で再開可能
- **停止シーケンス:**
  - 即時: PWM→0 (パルス停止)
  - 段階: 電流監視+ゲート駆動制御 (慣性停止)

### 6.4 MCU自己診断機構 (FSR-007) - IEC 60730 Annex H準拠

| 診断項目 | 検査対象 | 手法 | 周期 | 異常時動作 |
|---------|---------|------|------|----------|
| **CPU診断** | 汎用レジスタ | 全ビット読み書き | 起動時 | リセット→安全遷移 |
| **RAM診断** | 内部SRAM 12KB | March C+ | 起動+ランタイム | リセット→安全遷移 |
| **ROM診断** | Flash 256KB | CRC32 | 起動時 | リセット→安全遷移 |
| **クロック診断** | 内部/外部クロック | クロスモニタ | 継続監視 | WDTトリガ |

**実装フロー:**

```
電源ON
  ↓
[スタートアップ診断]
  ├─ CPU Reg Test .......... 1秒以内
  ├─ RAM March Test ........ 50ms
  ├─ ROM CRC Check ......... 500ms
  ├─ Clock Monitor ......... 動作確認
  └─ WDT初期化
  ↓
  PASS? → メイン処理へ / FAIL? → リセット
  ↓
[ランタイム診断]
  ├─ RAM分割March .......... 毎制御周期 (256B/サイクル)
  ├─ WDTキック ............ 毎制御周期
  └─ 異常検出 → 安全状態遷移実行
```

### 6.5 安全状態遷移メカニズム (FSR-008)

**異常検出トリガ:**
1. 自己診断失敗 (TSR-008~011)
2. WDTタイムアウト (TSR-015)
3. 過速度継続 (閾値超過 >3周期)
4. 過電流継続 (閾値超過 >3周期)

**安全状態遷移時の動作:**
```
異常検出
  ↓
[即座の出力停止]
  ├─ 全PWM出力 = 0 (即時)
  ├─ インバータゲート遮断 (HW信号)
  └─ 制御フロー中断
  ↓
[状態記録]
  ├─ エラーフラグ = 1
  ├─ エラーコード記録
  └─ 発生時刻タイムスタンプ
  ↓
[通知]
  ├─ LEDランプ点灯 (異常表示)
  ├─ ブザー出力 (オプション)
  └─ 外部制御盤への信号送信
  ↓
[停止状態維持]
  └─ 手動リセット待ち (再起動)
```

### 6.6 WDT監視メカニズム (TSR-015)

**概要:**
ハードウェアWDTを使用したファームウェア監視

**設定:**
- **WDTタイムアウト:** TBD (候補: 100ms~500ms)
- **キック周期:** メイン制御ループ完了後のみ
- **キック失敗:** MCU強制リセット→安全状態遷移

**キック管理フロー:**
```
制御ループ開始
  ├─ 入力取得 (センサ読み込み)
  ├─ 処理実行 (制御演算)
  ├─ 出力更新 (PWM/ゲート)
  ├─ 自己診断 (ランタイムテスト)
  ├─ エラーチェック
  │  └─ 異常あり? → 安全遷移 (WDTキックスキップ)
  ├─ 異常なし? → WDTキック実行 ✓
  └─ ループ周期満了まで待機

タイムアウト時: MCU自動リセット → ブートローダ → 起動診断 → 安全状態遷移
```

## 7. 電圧監視・管理メカニズム (FSR-009→010)

**概要:**
```
電源電圧 (分圧回路) → ADC変換 → デジタルフィルタ
  → 閾値判定 → 起動禁止制御
```

**技術詳細:**

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| **電源電圧範囲** | 5V (MCU VDD) ±10% | 4.5V ~ 5.5V |
| **監視下限** | 4.5V | 5V × 90% |
| **監視上限** | 5.5V | 5V × 110% |
| **ADC分圧** | VDD/2 | 2.75V (上限時) |
| **ADCサンプリング** | 1kHz | フィルタ前 |
| **デジタルフィルタ** | 移動平均 N=5 | 遅延 5ms |
| **判定周期** | 10ms | 診断実施周期 |

**監視ロジック:**
```
V_in (MCU VDD 5V)
  ↓ [分圧 1/2 (VDD/2→ADC入力)]
V_adc = V_in / 2
  ↓ [ADC変換]
D_raw (12ビット)
  ↓ [移動平均フィルタ]
D_filtered
  ↓ [閾値比較]
  ├─ D_filtered < 2250 (4.5V) ? → V_low = 1
  ├─ D_filtered > 2750 (5.5V) ? → V_high = 1
  └─ else → V_normal = 1
  ↓ [禁止フラグ制御]
  ├─ (V_low OR V_high) ? → mostart_inhibit = 1
  └─ else ? → mostart_inhibit = 0
```

**復帰条件:**
- 電圧が正常範囲に戻った状態で 200ms以上継続
- ユーザが手動リセットボタン押下
- 制御盤による外部リセット信号
