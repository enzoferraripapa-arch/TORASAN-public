# コーディングガイドライン

**プロジェクト：** 洗濯機モータ制御システム（IEC 60730 クラスB）
**適用範囲：** 全安全関連 C コード
**規格基準：** MISRA-C:2012 + IEC 60730 Annex H

---

## 1. 適用範囲

本ガイドラインは以下のコードに適用される：

- **対象コード：** SAP-ID SA-001～SA-008 （安全関連モジュール）
- **対象言語：** C言語（ISO/IEC 9899:1999）
- **非対象：** ユーティリティスクリプト、UI通信プロトコル、テストハーネス（既に別規約）

### 適用強度レベル

| レベル | 定義 | 該当モジュール |
|:--:|:--|:--|
| **レベルA（最高）** | 安全状態管理、異常検出ロジック | SA-006, SA-005 |
| **レベルB（高）** | モータ制御、監視ロジック | SA-002, SA-003, SA-004, SA-007, SA-008 |
| **レベルC（中）** | 初期化、UI通信 | SA-001, SA-012 |

## 2. 言語規格

### 2.1 採用規格

**標準:** ISO/IEC 9899:1999 (C99)

### 2.2 コンパイラ設定
```makefile
# RL78 GCC コンパイラ推奨設定
CFLAGS = -std=c99 \
         -Wall -Wextra -Werror \
         -pedantic \
         -fno-common \
         -fno-builtin \
         -fdata-sections -ffunction-sections \
         -O2
```

### 2.3 禁止言語機能

| 機能 | 理由 | 例外 |
|:--|:--|:--|
| 動的メモリ（malloc/free/realloc） | 動的動作の予測不可 | なし |
| 再帰関数 | スタック使用量の計算不可 | なし |
| 浮動小数点演算 | 丸め誤差の影響、規格未対応 | テストコードのみ可 |
| VLA（可変長配列） | メモリ使用量の動的変化 | なし |
| setjmp/longjmp | 状態管理の複雑化 | エラーハンドリングは明示的フロー |
| goto（一般） | コード流は分岐のみ | エラー処理の早期抜けは可 |
| ポインタ演算 | 境界外アクセスリスク | 配列インデックスで代替 |
| union型 | 型安全性の低下 | メモリ効率が必須の場合のみ許可 |

## 3. MISRA-C:2012 規約（RL78/RX安全開発向け）

### 3.1 必須ルール (Mandatory Rules)

#### 関数設計
- **R1.1:** すべての関数は単一エントリポイント・単一エクスポイント
  ```c
  // ✅ Good
  int ModuleName_ProcessData(uint16_t input) {
      int result = 0;
      if (input > MAX_VAL) {
          result = -1;
      } else {
          result = input * 2;
      }
      return result;
  }

  // ❌ Bad: 途中で複数のreturn
  int ModuleName_ProcessData(uint16_t input) {
      if (input > MAX_VAL) return -1;
      return input * 2;
  }
  ```

- **R2.1:** 無限ループは明示的制御フラグで管理
  ```c
  // ✅ Good: main loop with exit condition
  static volatile uint8_t exit_flag = 0;
  while (!exit_flag) {
      execute_task();
      if (shutdown_requested) {
          exit_flag = 1;
      }
  }

  // ❌ Bad: 無条件ループ
  while (1) { ... }
  ```

- **R5.1:** 外部スコープ変数の使用禁止（状態に直結）
  ```c
  // ❌ Bad
  static uint16_t g_speed = 0;
  void motor_ctrl_update() {
      g_speed++;  // 外部変数直接変更
  }

  // ✅ Good: getter/setter経由
  void motor_ctrl_set_speed(uint16_t speed) {
      // バリデーション必須
      if (speed <= MAX_SPEED) {
          g_motor_state.speed = speed;
      }
  }
  ```

#### 型安全性
- **R6.1:** 符号付き/符号なし型の混在禁止
  ```c
  // ❌ Bad
  uint16_t u_val = 100;
  int16_t s_val = -50;
  int result = u_val + s_val;  // 型が混在

  // ✅ Good: 明示的キャスト＋コメント
  uint16_t u_val = 100U;
  int16_t s_val = -50;
  uint16_t result = u_val + (uint16_t)s_val;
  ```

- **R7.1:** 配列境界チェック必須
  ```c
  // ✅ Good
  #define ARRAY_SIZE 256
  uint8_t buffer[ARRAY_SIZE];

  void buffer_write(uint16_t index, uint8_t data) {
      if (index < ARRAY_SIZE) {  // 必須チェック
          buffer[index] = data;
      }
  }

  // ❌ Bad: チェックなし
  void buffer_write(uint16_t index, uint8_t data) {
      buffer[index] = data;
  }
  ```

### 3.2 推奨ルール (Advisory Rules) - 安全開発向けサブセット

- **A1.1:** 関数は30行以内（複雑度CC15以下）
- **A2.1:** 変数は使用直前に初期化
- **A3.1:** コメント密度：最小10%行

## 4. IEC 60730 Annex H 固有規約

### 4.1 自己診断ルーチン実装規約

#### ソースコード自己診断（CRC）
```c
/**
 * @req IEC 60730:H.2.3
 * @asil QM
 *
 * プログラムメモリ CRC 検査ルーチン
 * 実行スケジュール: 起動時、周期テスト（100ms）
 */
static uint32_t safe_crc32_compute(const uint8_t *data, uint32_t length) {
    uint32_t crc = 0xFFFFFFFFUL;
    for (uint32_t i = 0; i < length; i++) {
        uint8_t byte = data[i];
        crc ^= byte;
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x80000000UL) {
                crc = (crc << 1) ^ 0x04C11DB7UL;
            } else {
                crc = crc << 1;
            }
            crc &= 0xFFFFFFFFUL;
        }
    }
    return crc ^ 0xFFFFFFFFUL;
}

// 起動時検査
void safe_rom_check_startup() {
    uint32_t calculated_crc = safe_crc32_compute(
        (uint8_t *)PROGRAM_START,
        PROGRAM_LENGTH
    );

    if (calculated_crc != STORED_CRC_VALUE) {
        SafetyMgr_ReportFault(FAULT_ROM_CRC_MISMATCH);
    }
}
```

#### メモリ自己診断（RAM）
```c
/**
 * @req IEC 60730:H.2.9
 * @asil Class B
 *
 * RAM マーチング メモリテスト（MATS）
 * 実行スケジュール: 周期テスト（100ms毎に16バイト）
 */
typedef struct {
    uint16_t test_index;      // テスト進行位置
    uint32_t pattern;         // テストパターン
    uint8_t test_phase;       // 0=W0, 1=R0, 2=W1, 3=R1
} RamTestState_t;

static RamTestState_t ram_test_state = {0};

void safe_ram_check_cyclic(void) {
    #define RAM_TEST_CHUNK_SIZE 16
    #define RAM_TEST_AREA_START ((uintptr_t)0x000F8000)
    #define RAM_TEST_AREA_END   ((uintptr_t)0x000F9000)

    volatile uint32_t *test_ptr =
        (volatile uint32_t *)(RAM_TEST_AREA_START + ram_test_state.test_index);

    switch (ram_test_state.test_phase) {
        case 0:  // Write 0x00000000
            for (int i = 0; i < RAM_TEST_CHUNK_SIZE/4; i++) {
                test_ptr[i] = 0x00000000UL;
            }
            ram_test_state.test_phase = 1;
            break;

        case 1:  // Read 0x00000000
            for (int i = 0; i < RAM_TEST_CHUNK_SIZE/4; i++) {
                if (test_ptr[i] != 0x00000000UL) {
                    SafetyMgr_ReportFault(FAULT_RAM_STUCK);
                    return;
                }
            }
            ram_test_state.test_phase = 2;
            break;

        case 2:  // Write 0xFFFFFFFF
            for (int i = 0; i < RAM_TEST_CHUNK_SIZE/4; i++) {
                test_ptr[i] = 0xFFFFFFFFUL;
            }
            ram_test_state.test_phase = 3;
            break;

        case 3:  // Read 0xFFFFFFFF
            for (int i = 0; i < RAM_TEST_CHUNK_SIZE/4; i++) {
                if (test_ptr[i] != 0xFFFFFFFFUL) {
                    SafetyMgr_ReportFault(FAULT_RAM_STUCK);
                    return;
                }
            }
            // 復旧
            for (int i = 0; i < RAM_TEST_CHUNK_SIZE/4; i++) {
                test_ptr[i] = 0x00000000UL;
            }
            // 進行
            ram_test_state.test_index += RAM_TEST_CHUNK_SIZE;
            ram_test_state.test_phase = 0;

            if (ram_test_state.test_index >=
                (RAM_TEST_AREA_END - RAM_TEST_AREA_START)) {
                ram_test_state.test_index = 0;  // 循環
            }
            break;
    }
}
```

#### 時間ベース監視（Clock Test）
```c
/**
 * @req IEC 60730:H.2.5
 * @asil Class B
 *
 * クロック周波数監視
 */
#define CPU_CLOCK_REF_COUNTS_10MS 160000  // @ 16MHz, 10ms相当

static uint16_t clock_error_counter = 0;

void safe_clock_check_on_timer_interrupt(void) {
    static uint16_t counter = 0;
    static uint32_t tick_count = 0;

    tick_count++;

    if (tick_count >= 10000) {  // 10ms毎
        // リファレンス時間ソース（RTC等）との比較
        // ここでは簡略化：期待値との比較
        if (counter < CPU_CLOCK_REF_COUNTS_10MS * 0.9 ||
            counter > CPU_CLOCK_REF_COUNTS_10MS * 1.1) {
            clock_error_counter++;
            if (clock_error_counter > 3) {  // 3回連続エラー
                SafetyMgr_ReportFault(FAULT_CLOCK_ABNORMAL);
            }
        } else {
            clock_error_counter = 0;
        }
        counter = 0;
        tick_count = 0;
    } else {
        counter++;
    }
}
```

### 4.2 安全変数保護方式

#### パターン1: 逆コピー保護（Inverse Copy）
```c
/**
 * 安全変数を逆コピーで多重化
 */
typedef struct {
    uint16_t value;       // 実値
    uint16_t inverted;    // ~value（反転）
} SafeVar_uint16_t;

#define SAFE_VAR_SET(var, val) \
    do { \
        (var).value = (val); \
        (var).inverted = ~(val); \
    } while(0)

#define SAFE_VAR_GET(var) \
    (((var).value ^ (var).inverted) == 0xFFFFUL ? (var).value : 0xFFFFUL)

#define SAFE_VAR_CHECK(var) \
    (((var).value ^ (var).inverted) == 0xFFFFUL)

// 使用例
SafeVar_uint16_t motor_speed;
SAFE_VAR_SET(motor_speed, 1000);

if (SAFE_VAR_CHECK(motor_speed)) {
    uint16_t speed = SAFE_VAR_GET(motor_speed);
} else {
    SafetyMgr_ReportFault(FAULT_DATA_CORRUPTION);
}
```

#### パターン2: CRC保護（Critical Buffer）
```c
#define SAFE_BUFFER_SIZE 64

typedef struct {
    uint8_t data[SAFE_BUFFER_SIZE];
    uint32_t crc32;
} SafeBuffer_t;

static uint32_t safe_crc32_buffer(const uint8_t *data, uint16_t len) {
    // CRC32計算（前述の実装参照）
    return safe_crc32_compute(data, len);
}

void safe_buffer_write(SafeBuffer_t *buf, const uint8_t *src, uint16_t len) {
    if (len > SAFE_BUFFER_SIZE) {
        SafetyMgr_ReportFault(FAULT_BUFFER_OVERFLOW);
        return;
    }

    // CPUの割込みを禁止（一貫性確保）
    uint16_t psw = __get_psw();
    __DI();

    memcpy(buf->data, src, len);
    buf->crc32 = safe_crc32_buffer(buf->data, len);

    // 割込み復元
    __set_psw(psw);
}

uint8_t safe_buffer_read(SafeBuffer_t *buf, uint8_t *dst, uint16_t len) {
    uint32_t calc_crc = safe_crc32_buffer(buf->data, len);

    if (calc_crc != buf->crc32) {
        SafetyMgr_ReportFault(FAULT_BUFFER_CRC_MISMATCH);
        return 0;
    }

    memcpy(dst, buf->data, len);
    return 1;
}
```

### 4.3 クリティカルセクション管理

```c
/**
 * @req IEC 60730:H.2.1
 * 割込みによる状態破壊からの保護

// RL78用マクロ（プラットフォーム依存）
#define SAFE_ENTER_CRITICAL() do { \
    uint16_t saved_psw = __get_psw(); \
```

## 7. コーディング例 - Complete Safety Module

```c
/**
 * @file       safety_mgr.c
 * @brief      安全管理モジュール実装
 * @req        SR-013
 * @asil       ASIL B
 */

#include <stdint.h>
#include <string.h>

// ============================================================================
// 型定義
// ============================================================================

typedef enum {
    SAFE_STATE_INIT = 0,
    SAFE_STATE_NORMAL = 1,
    SAFE_STATE_ERROR = 2,
    SAFE_STATE_SHUTDOWN = 3,
} SafeState_t;

typedef enum {
    FAULT_NONE = 0,
    FAULT_ROM_CRC_MISMATCH = 1,
    FAULT_RAM_STUCK = 2,
    FAULT_CLOCK_ABNORMAL = 3,
    FAULT_MOTOR_OVERCURRENT = 4,
    FAULT_TEMPERATURE_HIGH = 5,
    FAULT_LID_OPEN = 6,
    FAULT_VOLTAGE_LOW = 7,
    FAULT_VOLTAGE_HIGH = 8,
} FaultId_t;

// ============================================================================
// グローバル変数（安全保護）
// ============================================================================

/**
 * @safe       逆コピー保護
 * @corruption FAULT_STATE_CORRUPTED
 */
SafeVar_uint8_t g_safety_state;

/**
 * @safe       逆コピー保護
 * @corruption FAULT_FAULT_ID_CORRUPTED
 */
SafeVar_uint16_t g_fault_id;

static uint16_t fault_counter = 0;

// ============================================================================
// 関数実装
// ============================================================================

/**
 * @brief      安全管理モジュール初期化
 * @req        SR-013
 * @asil       ASIL B
 *
 * @return     0=成功, 1=失敗
 */
int8_t SafetyMgr_Init(void) {
    SAFE_VAR_SET(g_safety_state, (uint8_t)SAFE_STATE_INIT);
    SAFE_VAR_SET(g_fault_id, (uint16_t)FAULT_NONE);
    fault_counter = 0;

    return 0;
}

/**
 * @brief      異常報告インターフェース
 * @req        SR-013, SR-009, SR-010, SR-011, SR-012, SR-014, SR-015, SR-016
 * @asil       ASIL B
 *
 * @param[in]  fault_id 異常ID
 */
void SafetyMgr_ReportFault(FaultId_t fault_id) {
    SAFE_ENTER_CRITICAL();

    uint8_t current_state = SAFE_VAR_GET(g_safety_state);

    // 状態遷移
    if (current_state != (uint8_t)SAFE_STATE_ERROR &&
        !SAFE_VAR_CHECK(g_safety_state)) {
        // 状態破壊検出
        SAFE_EXIT_CRITICAL();
        SafetyMgr_EnterErrorState(FAULT_NONE);
        return;
    }

    SAFE_VAR_SET(g_fault_id, (uint16_t)fault_id);
    SAFE_VAR_SET(g_safety_state, (uint8_t)SAFE_STATE_ERROR);

    // モータ制御に停止を指令
    MotorCtrl_EmergencyStop();

    // WDT禁止状態に（キック禁止）
    // → WDT timeout → リセット

    SAFE_EXIT_CRITICAL();
}

/**
 * @brief      健全性チェック
 * @req        SR-013
 * @asil       ASIL B
 *
 * @return     1=健全, 0=異常
 */
uint8_t SafetyMgr_IsHealthy(void) {
    if (!SAFE_VAR_CHECK(g_safety_state) ||
        !SAFE_VAR_CHECK(g_fault_id)) {
        return 0;
    }

    uint8_t state = SAFE_VAR_GET(g_safety_state);
    return (state == (uint8_t)SAFE_STATE_NORMAL) ? 1 : 0;
}

/**
 * @brief      エラー状態遷移（内部用）
 */
static void SafetyMgr_EnterErrorState(FaultId_t fault_id) {
    SAFE_ENTER_CRITICAL();

    SAFE_VAR_SET(g_safety_state, (uint8_t)SAFE_STATE_ERROR);
    SAFE_VAR_SET(g_fault_id, (uint16_t)fault_id);
    MotorCtrl_EmergencyStop();

    SAFE_EXIT_CRITICAL();
}
```

## 8. 禁止パターンと代替方法

| 禁止パターン | 理由 | 代替方法 |
|:--|:--|:--|
| `malloc()/free()` | 動的メモリの予測不可 | 静的バッファ確保、リングバッファ |
| 無限ループ `while(1)` | 終了条件不明 | `while (!exit_flag)` + 明示的設定 |
| 浮動小数点 | 丸め誤差、ハード非対応 | 固定小数点演算（Q表記） |
| 再帰関数 | スタック使用不明 | 反復実装、状態機械 |
| 複数return | 終了処理の漏れリスク | result変数 + 単一exit |
