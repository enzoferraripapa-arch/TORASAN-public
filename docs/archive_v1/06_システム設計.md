### 2.2 信号タイミング仕様

#### PWM信号 (IF-001, IF-002, IF-003)
```
キャリア周波数: 20 kHz (周期 50 µs)
デューティ比: 0 ~ 100% (0.1% ステップ)
立上がり/立下がり時間: < 500 ns (ゲートドライバ依存)
スイッチング方式: Space Vector PWM (SVPWM)
ピン: P10 (TAU0-0), P11 (TAU0-1), P12 (TAU0-2)
```

#### Hall センサ入力 (IF-004, IF-005, IF-006)
```
信号形式: デジタル (3状態, 120度配置)
サンプリング: エッジトリガ割込 (Timer TAU0 ch4-6 capture)
期待される遷移パターン:
  - 回転時: 001 → 101 → 100 → 110 → 010 → 011 → 001 (繰返し)
  - 停止時: 不変
最大回転数: 1500 rpm (定格), 1800 rpm (過速度閾値+20%)
ピン: P30 (TAU0-4), P31 (TAU0-5), P32 (TAU0-6)
```

#### ADC入力 (IF-007, IF-008)
```
解像度: 10 bit (Vref=5.0V)
入力範囲: 0 ~ 5.0V (MCU基準電圧)
サンプリング周期:
  - 電流 (IF-007): 10 ms (100 Hz)
  - 電圧 (IF-008): 100 ms (10 Hz)
変換時間: ~ 4 µs (RL78/G14)
ピン: ANI0 (電流センサ), ANI1 (電圧監視)
```

#### GPIO信号 (IF-009, IF-010)
```
信号レベル: CMOS (0/3.3V)
入力 (IF-009): 蓋センサ（P43, pull-up）
  - 蓋閉: 0V (GND接続)
  - 蓋開: 3.3V (オープン)
  - チャタリング対策: 20 ms debounce
出力 (IF-010): 蓋ロックアクチュエータ（P44）
  - ロック: 3.3V (リレー ON)
  - 解除: 0V (リレー OFF)
```

#### UART通信 (IF-011)
```
フォーマット: 8N1 (8 bit, No Parity, 1 stop bit)
ボーレート: 9600 bps
プロトコル: Master-Slave (MCU=Master, Panel=Slave)
メッセージ形式:
  - コマンド: [CMD] [DATA0] [DATA1] [CHECKSUM] \r\n
  - レスポンス: [ACK/NAK] [DATA] [CHECKSUM] \r\n
タイムアウト: 100 ms (レスポンス待機)
ピン: TXD0/RXD0 (RL78/G14 UART0)
```

#### WDT (IF-012)
```
タイムアウト値: 100 ms
キック周期: 50 ms
キック方式: 専用レジスタへの書き込み
タイムアウト時: HW自動リセット → 起動時テスト実行
```

## 4. 安全機構の詳細設計

### 4.1 過速度保護 (Overspeed Protection)

**対象要件**: TSR-xxx (速度上限制御)
**ASIL**: B

**実装方式**:
```
[Hall Center Logic]
     ↓ (Period calculation)
[RPM Calculation] → RPM_calc
     ↓
[Threshold Comparison] → RPM_calc > RPM_limit ?
     ↓ (YES)
[Emergency Stop] → PWM = 0% (全相OFF)
     ↓
[Shutdown Sequence] → Gate drivers disable
     ↓
[Error Flag Set] → UI通知
```

**詳細アルゴリズム**:
1. Hall入力のエッジ間隔を測定 (Timer Capture, TAU0 ch4-6)
2. 回転周期 T_rot を計算
3. RPM_calc = 60,000 / (6 × T_rot [ms])
4. RPM_calc > RPM_limit (1500 rpm) → 過速度判定
5. すべてのPWM出力を0% に設定 (強制OFF)
6. ゲートドライバの出力 (UH/UL/VH/VL/WH/WL) を全て無効化
7. エラーフラグ (ERR_OVERSPEED) をセット
8. UI パネルへエラーコード送信
9. リセットまで回転禁止状態を維持

**時間要件**:
- 過速度判定～PWM OFF: < 10 ms (1 Hall周期想定)
- 完全停止時間: < 500 ms (蓋ロック確保)

---

### 4.2 過電流保護 (Overcurrent Protection)

**対象要件**: TSR-xxx (電流制限)
**ASIL**: B

**実装方式**:
```
[ADC 100Hz Sampling] → I_raw (Current sensor ADC value)
     ↓
[Digital Low Pass Filter (LPF)]
     I_filt = 0.9 × I_filt + 0.1 × I_raw
     ↓
[Threshold Comparison] → I_filt > I_limit ?
     ↓ (YES, consecutive 2 checks)
[Current Limit Control] → Reduce PWM duty
     ↓ (Still exceeds after control)
[Emergency Shutdown] → PWM = 0%, Gate OFF
     ↓
[Relay Shutdown] → Power relay OFF
```

**詳細アルゴリズム**:
1. ADC Channel 0 (ANI0) から電流センサ値を連続 100 Hz サンプリング
2. デジタルLPF適用 (カットオフ ~100 Hz)
3. フィルタ後値 I_filt と閾値 I_limit (8A) を比較
4. I_filt > I_limit を2回連続検出 → 過電流状態
5. PWM デューティ を -10% (速度低下)
6. 3回目検出で → PWM = 0% (全相OFF)
7. ゲートドライバ完全無効化
8. 電源リレー OFF (リセット)
9. エラーログ記録 (I_peak値, 継続時間)

**時間要件**:
- LPF応答時間: ~ 3 ms
- 過電流判定～初期制御: < 5 ms
- 完全遮断: < 20 ms

---

### 4.3 蓋インターロック (Door Interlock)

**対象要件**: TSR-xxx (蓋ロック制御)
**ASIL**: A (safety-critical)

**実装方式**:
```
[GPIO 100Hz Polling] → DoorState_raw
     ↓
[Debounce Logic (20ms)]
     count_raw = (DoorState_raw == prev) ? count_raw + 1 : 0
     if (count_raw >= 2) → DoorState_stable = DoorState_raw
     ↓
[State Machine]
  (CLOSED, LOCKED) ← Normal run state
       ↓
     (DoorState_stable == OPEN)
       ↓
  (OPEN, UNLOCK_REQ)
       ↓
  [Motor Stop Sequence]
    - PWM_duty → 0%
    - Phase short (optional)
    - Wait for stop detection
       ↓
  (OPEN, UNLOCKING)
       ↓
  [Unlock Actuator]
    - GPIO_LOCK_CTRL = 0V (3 ms pulse)
       ↓
  (OPEN, UNLOCKED) → User can open
```

**詳細アルゴリズム**:

| 状態 | 入力 | 動作 | 次状態 |
|-----|------|------|--------|
| CLOSED_LOCKED | 蓋開信号（P43） | モータ停止開始 | OPEN_STOP_REQ |
| OPEN_STOP_REQ | RPM=0 (確認) | ロック解除指令 | OPEN_UNLOCKING |
| OPEN_UNLOCKING | GPIO出力完了（P44→0V, 3ms後） | ロック状態維持 | OPEN_UNLOCKED |
| OPEN_UNLOCKED | 蓋閉信号 | ロック再施錠 | CLOSED_LOCKING |
| CLOSED_LOCKING | ロック完了確認 | 運転再開可能 | CLOSED_LOCKED |

**チャタリング対策**:
- サンプリング周期: 10 ms (100 Hz)
- デバウンス時間: 20 ms (2サンプル × 10 ms)
- 確定後のヒステリシス: 30 ms (運転開始前の安定時間確保)
- GPIO：P43（入力、内部プルアップ）、P44（出力）

**タイムアウト保護**:
- ロック解除完了タイムアウト: 500 ms
  - 期限内に蓋開信号が確認されない場合 → エラー状態 → リセット待機

**詳細テスト項目**:

| テスト項目 | 方法 | 周期 | 許容値 |
|----------|------|------|-------|
| CPU Register | Walking bit / Checkerboard pattern | Start | 100% correct |
| RAM (Full) | March C algorithm | Start | 100% coverage |
| ROM CRC32 | CRC32 (IEEE 0x04c11db7)計算＆事前値比較 | Start | Match expected |
| Clock Freq | Timer周波数計測 | Start | ±5% nominal |
| ADC Offset | Self-calibration (ADC0/1) | Start + 1Hz | ±10 LSB |
| GPIO Port | Write/Read test | Start | Bit-by-bit match |
| Timer Module | Counter increment | Start | Monotonic increase |

#### 4.4.2 ランタイム定期テスト (Runtime Periodic Tests)

```
Main Loop (50ms cycle)
     ↓
  [Periodic Task Scheduler]
     ├─ Every 50 ms: CRC32 (Program ROM range check, 4KB block)
     ├─ Every 100 ms: ADC Range check (0-1023, 10bit)
     ├─ Every 500 ms: Clock Frequency verify
     ├─ Every 1 sec: Watchdog heartbeat
     └─ Every 10 sec: Full RAM walk test (limited)
```

**実装例: CRC32 (IEEE 0x04c11db7)チェック (ランタイム)**
```
ROM領域を4KBブロック単位に分割
毎サイクル異なるブロックのCRC32を計算
前回計算値と比較 → 不一致 → ERR_ROM_CORRUPT フラグセット
→ 次ループ: PWM = 0%, シャットダウン開始
```

### 4.5 ウォッチドッグタイマ (WDT)

**対象要件**: TSR-xxx (MCU監視)
**ASIL**: B

**実装方式**:
```
[Main Loop Cycle]
     ↓ (50 ms)
┌────────────────────┐
│ Task 1: Hall       │
│ process           │
│ ~5 ms             │
└────────┬───────────┘
         ↓
┌────────────────────┐
│ Task 2: Speed      │
│ control           │
│ ~10 ms            │
└────────┬───────────┘
         ↓
┌────────────────────┐
│ Task 3: Current    │
│ protection        │
│ ~8 ms             │
└────────┬───────────┘
         ↓
┌────────────────────┐
│ Task 4: Safety     │
│ diagnosis         │
│ ~10 ms            │
└────────┬───────────┘
         ↓
┌────────────────────┐
│ Task 5: UI UART    │
│ communication     │
│ ~12 ms            │
└────────┬───────────┘
         ↓
┌─────────────────────────────┐
│ WDT Kick (< 50 ms)         │
│ WDT.REG = 0xAC             │
│ (Dummy write pattern)       │
└─────────────────────────────┘
         ↓
   Total: ~45 ms < 50 ms OK
         ↓
   [Loop Back] (50 ms cycle)
```

**WDT設定**:
- タイムアウト値: 100 ms
- キック周期: 50 ms (安全係数 = 2)
- キック失敗回数: 1回 → タイムアウト検出
- タイムアウト時動作: HW自動リセット

**タイムアウト発生時**:
1. MCU自動リセット (< 1 µs)
2. 起動時テスト実行 (テスト失敗なら halt)
3. 正常完了後メインループ再開
4. UI パネルへエラーコード送信

**キック失敗検出メカニズム**:
```
Loop iteration counter:
  - Global counter (RAM内)
  - 毎ループで +1
  - WDT Kick成功時、タイムスタンプ記録
  - 次ループでカウンタ進行確認
  - 5ループ以上進行なし → Hang up 判定
  - 診断エラーログ出力
```

### 4.6 電圧監視 (Voltage Supervision)

**対象要件**: TSR-xxx (電源管理)
**ASIL**: B

**実装方式**:
```
[ADC 1kHz Sampling] → V_raw (DC Bus voltage)
     ↓
[LPF Filtering (cutoff 50Hz)]
     V_filt = 0.95 × V_filt + 0.05 × V_raw
     ↓
[Window Comparator]
     ├─ V_filt < V_low (44V) → Undervoltage
     ├─ V_high < V_filt < V_low → Normal
     └─ V_filt > V_high (56V) → Overvoltage
     ↓
[Action]
 Undervolage: Stop operation, wait for recovery, timeout 5 sec → shutdown
 Overvoltage: Stop operation immediately, relay OFF, UI error
 Normal: Continue operation
```

**電圧レンジ定義**:
```
Nominal: 48V ± 10% = 43.2 ~ 52.8V
Safe operation window:
  V_low_warning  = 45V  (−6.25%)  ← 動作継続可能下限
  V_high_warning = 54V  (+12.5%)  ← 動作継続可能上限
  V_low_shutdown = 44V  (−8.3%)   ← 強制停止下限
  V_high_shutdown= 56V  (+16.7%)  ← 強制停止上限
```

**タイムアウト機構**:
```
Undervoltage detected:
  ├─ Timer start (Recovery window 5 sec)
  ├─ Timer running: PWM gradual reduction
  └─ Timer timeout: PWM = 0%, Gate OFF, Relay OFF

Recovery:
  ├─ V_filt returns to V_low_warning range
  └─ Timer cancel, normal operation resume
```

## 6. 実装検討事項

### 6.1 PWM駆動方式の選択

**オプション A: 単相PWM (1相制御)**
- 簡易実装
- BLDC不適切 → 脱落するセクタあり

**オプション B: Sinusoidal PWM (正弦波PWM)**
- 標準的な3相駆動
- リップル大きい
- 演算負荷 中程度

**オプション C: Space Vector PWM (SVPWM) ← 推奨**
- リップル最小化
- DC Bus利用率 最大化 (−15.47% vs Sinusoidal)
- 演算負荷 高い (但しRL78で可能)
- 選定理由: 効率 + 低騒音 + 低EMI

---

### 6.2 デジタルLPFの実装

**IIR 1次 LPF** (推奨, 軽量):
```
y[n] = α × y[n-1] + (1-α) × x[n]

例: 電流センサ (目標カットオフ 500Hz @ fs=10kHz)
  α = 0.9 × y[n-1] + 0.1 × x[n]
  fc ≈ 500 Hz
```

**FIR フィルタ** (高精度, 重い):
```
平均値フィルタ (8-tap):
  y[n] = (x[n] + x[n-1] + ... + x[n-7]) / 8
  遅延: 40 µs (4 samples × 10 µs)
  演算量: 8乗算 + 加算
```
