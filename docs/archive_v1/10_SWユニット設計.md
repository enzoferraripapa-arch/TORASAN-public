## 2. 各ユニットの詳細設計

### 2.1 MotorCtrl_GetRpm (UT-002)

```c
/**
 * @brief  現在の回転数を算出する
 * @req    SR-001 (モータ回転数の周期的監視)
 * @safety_class  クラスB
 * @test   TC-002 (TBD)
 *
 * @description
 *   パルスエンコーダまたはHallセンサからの入力を集計し、
 *   低速フィルタ(LPF)を適用して回転数を算出する。
 *   安全性：複数の計測周期にわたる移動平均により、
 *   ノイズ耐性および異常検出遅延の最小化を実現。
 *
 * @return 現在の回転数 [rpm] (0-20000)
 */
uint16_t MotorCtrl_GetRpm(void);
```

**実装ポイント:**
- Timer割込みによるパルスカウント（周期10ms）
- 移動平均フィルタ（直近3期間の平均）
- overflow防止：uint16_t範囲内で安全に処理

### 2.2 MotorCtrl_CheckOverspeed (UT-003)

```c
/**
 * @brief  過速度を判定する
 * @req    SR-002 (過速度の検出と即座の停止)
 * @safety_class  クラスB
 * @test   TC-003 (TBD)
 *
 * @description
 *   現在の回転数が設定された上限値を超えた場合、即座に警告/停止
 *   を発行する。デバウンス処理により、一過的なノイズを無視。
 *
 * @param  currentRpm  現在の回転数 [rpm]
 * @return SafetyStatus_t: OVERSPEED_DETECTED / NORMAL
 */
SafetyStatus_t MotorCtrl_CheckOverspeed(uint16_t currentRpm);
```

**アルゴリズム:**

```
入力: currentRpm

IF currentRpm > OVERSPEED_THRESHOLD (例: 1500 rpm) THEN
  overspeed_counter++
  IF overspeed_counter >= DEBOUNCE_COUNT (例: 3回) THEN
    SafetyMgr_TransitionSafe(FAULT_OVERSPEED)
    RETURN OVERSPEED_DETECTED
  ENDIF
ELSE
  overspeed_counter = 0
ENDIF

RETURN NORMAL
```

**安全根拠 (IEC 60730 Annex H):**
- **動的診断:** 回転数監視により、制御不可能な加速を即座に検出
- **デバウンス:** 誤検出防止（ノイズに強い）
- **多重性:** 複数の監視層（過速度 + WDT + 電圧）で冗長性確保

### 2.10 SafeDiag_RamTest (UT-011)

```c
/**
 * @brief  RAM全体を March C アルゴリズムでテストする
 * @req    SR-010 (RAM障害の検出)
 * @safety_class  クラスB
 * @test   TC-011 (TBD)
 *
 * @description
 *   IEC 60730 Annex H.3.2: Static Memory Test (March C)
 *   March C: 6N+4のメモリアクセスで以下を検出：

 *     - coupling fault (ビット間の相互作用)

DiagStatus_t SafeDiag_RamTest(void);
```

**March C のパス:**
```
1. Ascending  Write 0, Read 0
2. Ascending  Write 1, Read 1, Write 0, Read 0
3. Descending Write 1, Read 1, Write 0, Read 0
4. Descending Write 1, Read 1
```

## 3. 安全関連ユニットの設計根拠

### 3.1 診断アーキテクチャ

IEC 60730 Annex H 「Electronic Safety Systems」に準拠した
多層診断システムを採用：

**層1: 起動時自己診断 (Startup Self-Diagnostics)**
```
main() 起動直後に実行：
  [UT-010] CPU Register Test
  [UT-011] RAM March C Test
  [UT-012] ROM CRC32 Test
  [UT-013] Clock Test

  いずれか1つが失敗 → SafetyMgr_TransitionSafe()
           → システム永久ロック (再スタート不可)
```

**層2: 周期的ランタイム診断 (Periodic Runtime Diagnostics)**
```
メインループ内：
  [10ms周期]  UT-002, UT-003, UT-005, UT-006, UT-008
              → 制御対象の監視

  [100ms周期] UT-013 (Clock Check)
             UT-015, UT-016 (Voltage monitoring)
             → ハードウェア周辺の監視
```

**層3: イベント駆動型診断 (Event-Driven Diagnostics)**
```
割込みコンテキストから：
  [Timer ISR]     リアルタイム性要求
  [WDT ISR]       フロー異常即座検出
  [Fault ISR]     外部センサ異常
```

### 3.2 マルチレイヤ冗長性

**モータ停止コマンド:**
```
① MotorCtrl_EmergencyStop()  (PWM=0)
② CurrentMon_Shutdown()      (リレー遮断)
③ WDT timeout → Hardware リセット (最終手段)
```

**電源監視:**
```
① VoltageMon_CheckWindow()   (ソフトウェア監視)
② BOR (Brown-Out Detect)     (ハードウェア監視)
③ WDT timeout
```

### 3.3 デバウンス戦略

**目的:** 機械的接点ノイズ、ADCノイズ、通信エラーから
        安全機能を保護する。

**実装:**

| ユニット | デバウンス時間 | 連続判定回数 | 周期 | 総遅延時間 |
|---------|--------------|----------|------|----------|
| UT-003 (過速度) | 30ms | 3回 | 10ms | 30ms |
| UT-006 (過電流) | 20ms | 2回 | 10ms | 20ms |
| UT-008 (蓋開放) | 20ms | 3回 | 10ms | 30ms |

**許容遅延の確認:**
- 洗濯機の動作タイムスケール: 秒単位
- 診断遅延: 最大30ms → 十分なマージン

### 3.4 エラーハンドリング戦略

**エラーコード定義 (FaultCode_t):**

```c
typedef enum {
    FAULT_NONE           = 0x00,
    FAULT_CPU_REG        = 0x01,  /* CPU Register test fail */
    FAULT_RAM            = 0x02,  /* RAM March C fail */
    FAULT_ROM            = 0x03,  /* ROM CRC mismatch */
    FAULT_CLOCK          = 0x04,  /* Clock freq out of range */
    FAULT_OVERSPEED      = 0x05,  /* Motor overspeed detected */
    FAULT_OVERCURRENT    = 0x06,  /* Overcurrent detected */
    FAULT_VOLTAGE_LOW    = 0x07,  /* Supply voltage too low */
    FAULT_VOLTAGE_HIGH   = 0x08,  /* Supply voltage too high */
    FAULT_LID_OPEN       = 0x09,  /* Lid open during operation */
    FAULT_WDT_TIMEOUT    = 0x0A,  /* Watchdog timeout (hardware) */
} FaultCode_t;
```

**安全状態の定義:**
```
モータ: 回転停止
リレー: OFF (通電遮断)
LED: 赤色点灯 (警告表示)
再スタート: 禁止（電源再投入まで）
```

## 5. インターフェース定義

### 5.1 安全クラス B ユニット間通信プロトコル

**原則:**
- 全安全関連関数呼び出しは戻り値チェック必須
- 例外: void型函数でも内部エラー状態をフラグで保持

**例 (UT-003):**
```c
// 呼び出し側
SafetyStatus_t status = MotorCtrl_CheckOverspeed(current_rpm);
if (status == OVERSPEED_DETECTED) {
    // SafetyMgr_TransitionSafe() は内部で自動呼び出し済み
    // ここでは追加アラート等を実装
}
```

### 5.2 グローバル状態管理

**システム全体の状態フラグ:**
```c
typedef struct {
    uint8_t  system_state;      /* STARTUP_DIAG / NORMAL / SAFE_STATE */
    uint16_t fault_code;        /* 最後に検出された故障コード */
    uint32_t runtime_ms;        /* システム稼働時間 */
    uint8_t  diag_phase;        /* 起動時診断フェーズ (0-4) */
} SystemContext_t;

volatile SystemContext_t g_sysCtx = {0};
```
