# ツール適格性記録
## 洗濯機モータ制御システム（WMC）開発プロジェクト

---

## 1. ツール適格性管理の目的

本文書は、IEC 60730 クラスB 開発における信頼度レベル（TCL: Tool Confidence Level）に基づき、使用するツールの適格性を管理するものです。

**適用規格：** IEC 60730-1 Annex H（ソフトウェア要件）
**ツール管理基準：** TCL1, TCL2, TCL3（3段階評価）

### TCL の定義

| TCL | 説明 | 適用ツール | 必要な確認 |
|-----|------|----------|---------|
| **TCL1** | 低信頼度 | IDE、デバッガ、テストツール | 既知バージョン使用実績確認 |
| **TCL2** | 中信頼度 | コンパイラ、ビルドツール | ベンダ適格性証明 + 検証 |
| **TCL3** | 高信頼度 | 自動生成ツール（AI含む）、コード生成ツール | **全出力に対する独立検証が必須** |

## 3. ツール個別適格性評価

### 3.1 Claude Code（T-001）

#### 概要
- **分類：** AI ベース自動生成ツール
- **機能：** 技術仕様書生成、ソースコード生成、テストケース生成、ドキュメント作成
- **TCL：** **TCL3**（高信頼度、すべての出力に人間による検証が必須）

#### TCL3 ツール適格性確認方法（方法A：全成果物レビュー方法）

AI ツール（Claude Code）の出力物は、以下の徹底的なレビュー方法により適格性を確認します：

**A1. 機能安全適合性レビュー**
- IEC 60730 Annex H への準拠性確認
- 自動診断機能仕様の妥当性確認
- エラーハンドリング・異常検出の完全性確認

**A2. コード品質レビュー**
- MISRA C 準拠性確認（TC1002: MISRA コンプライアンス）
- メモリリーク、バッファオーバーフロー検出
- 静的解析ツール結果の確認

**A3. テスト充足性レビュー**
- テストケースの要件カバレッジ確認（最低 90% 目標）
- 診断機能テストの完全性確認
- エッジケース・エラーケースの網羅性確認

**A4. ドキュメント一貫性レビュー**
- 設計書・実装・テストの一貫性確認
- トレーサビリティ確認（要件→設計→コード→テスト）
- 誤字脱字・論理矛盾の検出

**A5. 安全機能適格性確認**
- モータ制御タイミング・精度確認
- センサ入力処理の安全性確認
- 診断エラーハンドリングの適切性確認

#### 出力成果物の確認品目

| 成果物 | 確認項目 | レビュー者 | 承認 |
|--------|--------|----------|------|
| 機能安全仕様書（SRS） | IEC 60730 適合性、要件の明確性、診断仕様 | TORA | □ |
| ソフトウェア設計書（SDS） | アーキテクチャ、モジュール設計、インターフェース | TORA | □ |
| ソースコード | MISRA C 準拠、エラーハンドリング、自動診断実装 | TORA | □ |
| テスト計画書 | カバレッジ目標、テストケース充足性、診断検証 | TORA | □ |
| テスト結果報告書 | カバレッジ達成、テスト合格、缺陥トレーサビリティ | TORA | □ |
| 設計・検証ドキュメント | トレーサビリティ行列、設計根拠、論理妥当性 | TORA | □ |

#### 適格性確認予定日・スケジュール

- **PH-04（機能安全仕様）：** SRS ドキュメント確認 → 予定日：TBD
- **PH-06（ソフト設計）：** SDS + テスト計画確認 → 予定日：TBD
- **PH-07（ソフト実装）：** ソースコード確認 → 予定日：TBD
- **PH-08（統合テスト）：** テスト結果確認 → 予定日：TBD
- **PH-12（形式評価）：** 全成果物統合レビュー → 予定日：TBD

#### 制限事項・注意点

1. **AI 生成コードの検証は必須：** TCL3 ツール出力は人間によるコード審査が絶対条件
2. **バージョン管理：** Claude Code のバージョン変更時は再確認が必要な場合がある
3. **安全機能の自動生成：** 自動診断機能はレビュー後、単体テストで必ず検証する
4. **独立検証：** CERTIFY モード移行時は外部専門家による独立検証を追加実施

### 3.2 Renesas CC-RL コンパイラ（T-002）

#### 概要
- **分類：** C 言語コンパイラ
- **機能：** RL78/RX アーキテクチャ用 C コード → アセンブリ/オブジェクトコード生成
- **TCL：** **TCL2**（中信頼度、ベンダ適格性評価 + 検証が必要）
- **ベンダ：** Renesas Electronics Corporation

#### TCL2 ツール適格性確認方法（方法B：ベンダ適格性評価）

**B1. ベンダ適格性確認**
- Renesas 正規ドキュメント入手・確認
- コンパイラリリースノート確認
- 既知の不具合・制限事項確認

**B2. コンパイラオプション検証**
- 最適化レベル（-O2, -O3）の安全性確認
- 診断オプション（-Wall, -Wextra）の有効化
- インラインアセンブリの正確性確認

**B3. テストコンパイル実行**
- サンプルコード（診断関数を含む）のコンパイル
- オブジェクトコード生成の正確性確認
- リンク・実行ファイル生成確認

**B4. バージョン管理**
- コンパイラバージョンの固定化
- ビルドスクリプトへのバージョン明記
- ビルド再現性の確保

#### 確認状態・スケジュール

| 確認項目 | 確認方法 | 予定日 | 確認者 |
|--------|--------|-------|-------|
| ベンダ技術情報確認 | 公式ドキュメント | PH-03 | TORA |
| コンパイラ動作確認 | テストコンパイル実行 | PH-03 | TORA |
| 適格性決定 | 最終承認 | PH-03 | TORA |

#### 利用条件

- **使用バージョン：** CC-RL ver. X.XX（TBD）
- **最適化レベル：** -O2 推奨（-O3 は検証後に許可）
- **診断オプション：** -Wall -Wextra 必須
- **ビルド記録：** コンパイラバージョン + オプションをビルドログに記録

### 3.3 Renesas e2studio IDE（T-003）

#### 概要
- **分類：** 統合開発環境（IDE）
- **機能：** コード編集、プロジェクト管理、デバッグ、シミュレータ実行
- **TCL：** **TCL1**（低信頼度、既知バージョンの使用実績確認のみ）
- **ベンダ：** Renesas Electronics Corporation

#### TCL1 ツール適格性確認方法（方法C：既知バージョン確認）

**C1. 既知バージョン使用確認**
- e2studio の安定バージョン使用
- サポート終了バージョンの回避
- RL78/RX サポートの確認

**C2. 動作確認**
- 開発ボード接続確認
- デバッガ（E1/E2）認識確認
- シミュレータ動作確認

**C3. バージョン記録**
- 使用 IDE バージョン記録
- ビルドログに IDE 情報を含める

#### 確認状態・スケジュール

| 確認項目 | 確認方法 | 予定日 | 確認者 |
|--------|--------|-------|-------|
| e2studio インストール | セットアップ実行 | PH-03 | TORA |
| バージョン確認 | ヘルプ→バージョン情報確認 | PH-03 | TORA |
| 開発ボード認識 | デバッガ接続テスト | PH-03 | TORA |

#### 利用条件

- **使用バージョン：** e2studio ver. X.XX（TBD）
- **デバッガ：** E1 または E2 エミュレータ
- **ビルド環境：** GNU make / CMake
- **継続性：** IDE 変更時は影響分析実施
