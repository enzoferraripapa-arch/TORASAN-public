# SWアーキテクチャ設計書

**プロジェクト：** 洗濯機モータ制御システム（IEC 60730 クラスB）
**適用MCU：** RL78/RX ファミリ
**アーキテクチャ：** スーパーループ（非RTOS）

---

## 1. SWアーキテクチャ概要

本プロジェクトは、スーパーループ型の非RTOS アーキテクチャを採用します。IEC 60730 クラスB の安全要件に対応し、モータ制御層と安全監視層を明確に分離した多層構造となります。

```
┌─────────────────────────────────────────────────┐
│              SWアーキテクチャ構成図                │
│                                                  │
│  ┌─────────┐   ┌──────────────────────────┐     │
│  │main()   │──→│ スーパーループ             │     │
│  │初期化   │   │ 1ms周期実行               │     │
│  └─────────┘   │                           │     │
│                │ ┌────────┐  ┌──────────┐ │     │
│                │ │モータ   │  │安全監視   │ │     │
│  ┌─────────┐   │ │制御層   │  │   層     │ │     │
│  │スタート │   │ │        │  │          │ │     │
│  │アップ   │   │ │SR-001〜│  │SR-009〜  │ │     │
│  │自己診断 │   │ │SR-008  │  │SR-016    │ │     │
│  └─────────┘   │ └────────┘  └──────────┘ │     │
│                │ ┌────────┐  ┌──────────┐ │     │
│                │ │センサ   │  │通信      │ │     │
│                │ │入力層   │  │  層      │ │     │
│                │ └────────┘  └──────────┘ │     │
│                └──────────────────────────┘     │
│  ┌──────────────────────────────────────────┐   │
│  │     HAL（ハードウェア抽象化層）             │   │
│  │  Timer / ADC / GPIO / UART / WDT           │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### アーキテクチャの特徴

- **スーパーループ型：** タイマ割込みによる1msティック駆動、メインループ内でタスク化されたポーリング処理を実行
- **多層分離設計：** モータ制御層と安全監視層を独立させ、安全機構の信頼性を確保
- **割込みベース：** 時間クリティカルな処理（PWMキャリア100μs、ADC変換）は割込みで駆動
- **ポーリング：** 非クリティカルな監視処理（電圧監視、自己診断）はメインループでポーリング実行
- **RAM保護：** 安全変数への逆コピー、CRC保護を実装

## 3. タスク・割り込み構成（スーパーループ）

### メインループ周期: 1ms（タイマ割込み駆動）

| 周期 | 処理内容 | 対応モジュール | 駆動方式 | 優先度 |
|:--:|:--------|:----------:|:------:|:---:|
| 100μs | Timer割込み（PWMキャリア同期） | SA-009 | 割込み | 最高 |
| 100μs | ADC変換完了割込み | SA-010 | 割込み | 高 |
| edge | ホールセンサ割込み | SA-009 | 割込み | 高 |
| 1ms | メインループティック、任意精度タスク実行 | SA-001 | ポーリング | 中 |
| 10ms | 電流監視＆モータ制御更新 | SA-002, SA-003 | ポーリング | 中 |
| 10ms | 蓋センサ監視 | SA-004 | ポーリング | 中 |
| 100ms | 電圧監視 | SA-007 | ポーリング | 低 |
| 100ms | ランタイム自己診断（分割実行） | SA-005 | ポーリング | 低 |
| 1000ms | WDTキック（安全タスク完了後） | SA-008 | ポーリング | 中 |
| 1000ms | UI通信処理 | SA-012 | ポーリング | 低 |

### タスク実行フロー（1msごと）
```
割込みコンテキスト:
  ├─ Timer割込み (100μs)
  │  └─ PWMキャリア更新
  ├─ ADC完了割込み (100μs)
  │  └─ ADC結果取得
  └─ ホール割込み (edge)
     └─ 回転数検出

メインループコンテキスト (1ms周期):
  ├─ 10ms: モータ制御更新
  ├─ 10ms: 電流監視
  ├─ 10ms: 蓋監視
  ├─ 100ms: 電圧監視
  ├─ 100ms: 自己診断（ステップ分割）
  ├─ 1000ms: WDT監視＆キック
  └─ 1000ms: UI通信
```

## 5. 安全機構の実装方針

### 5.1 安全監視層の独立性
- **安全監視層（SA-005, SA-006, SA-008）** はモータ制御層から明確に独立
- 安全状態の判定と制御をsafety_mgrに一元化
- モータ制御層がいかなる異常でも、安全層による停止指示を優先実行

### 5.2 異常検出と応答フロー
```
異常検出（各モジュール）
    ↓
SafetyMgr_ReportFault() 呼出
    ↓
安全状態遷移（正常 → エラー）
    ↓
全PWM出力停止（motor_ctrl）
    ↓
安全状態保持（WDTキック禁止）
    ↓
ユーザ通知（UI通信）
```

### 5.3 WDTキック制御
- WDTキックは **safety_mgrの完了確認後のみ** 実行
- 自己診断、異常監視、蓋チェックがすべて正常を確認した場合のみキック
- キック失敗時は即座に安全状態に遷移

### 5.4 割込み内での安全処理
- 割込みハンドラ内では **フラグ設定のみ** 行う
- 実際の安全判定はメインループで実行（状態の一貫性を確保）
- クリティカルセクション保護: __DI()/__EI() マクロ使用

## 6. メモリマップ概要

```
RL78/RX MCU メモリマップ（暫定）

フラッシュROM:
  0x00000000 - 0x0001FBFF : プログラム領域（128KB - 1KB）
  0x0001FC00 - 0x0001FFFF : CRC/シグネチャ領域（1KB）

内部RAM:
  0x000F8000 - 0x000F8FFF : グローバル変数・安全変数（4KB）
  0x000F9000 - 0x000FCFFF : スタック領域（16KB）
  0x000FD000 - 0x000FFFFF : ヒープ（未使用、予約）

SFR領域:
  0xFFF00000 - 0xFFFFFFFF : ペリフェラルレジスタ

※ MCU型番確定後、詳細メモリマップを更新
```

### メモリ使用見積り
| 区分 | 容量 | 備考 |
|:--|:--:|:--|
| プログラム | ~50KB | コード+定数 |
| グローバル変数 | ~2KB | 安全変数含 |
| スタック | ~4KB | 見積り（最終は静的解析で確認） |
| 合計 | ~56KB | 全体の44%（マージン確保） |

## 7. コーディングガイドライン適用方針

本プロジェクトの全安全関連コードは以下ドキュメントに従う：

**参照:** `docs/コーディングガイドライン.md`

### 主要ガイドライン
- **言語規格:** C99（ISO/IEC 9899:1999）
- **規約標準:** MISRA-C:2012（必須ルール）+ IEC 60730 Annex H
- **命名規則:** Module_Action型関数名、camelCase変数名、UPPER_SNAKE_CASE定数
- **安全タグ:** @req, @asil, @test Doxygen形式コメント
- **禁止事項:** malloc/free、goto、浮動小数点（安全コード）、無限ループ
