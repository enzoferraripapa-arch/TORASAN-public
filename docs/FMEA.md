# 故障モード影響解析（FMEA）
## 洗濯機BLDCモータ制御システム

文書番号：WMC-FMEA-001 / 版数：1.0 / 作成日：2026-03-01
SPICE: SYS.5 / ISO 26262: Part 5 §7 / ASIL: B

---

## §1. 概要

| 項目 | 内容 |
|------|------|
| 対象システム | 洗濯機BLDCモータ制御ECU（WMC-SD-001） |
| 解析目的 | 安全関連故障モードの同定、影響評価、対策の妥当性検証 |
| 解析範囲 | センサ入力系、MCU内部、アクチュエータ出力系の故障モード |
| 上位文書 | HARA（WMC-HARA-001）、FSC（WMC-FSC-001）、TSC（WMC-TSC-001）、システム設計書（WMC-SD-001） |
| 故障モード数 | 10件（FM-001〜FM-010） |
| 安全目標 | SG-001〜SG-007 |

### RPN評価基準

**重大度（S: Severity）**: 1〜10

| レベル | 説明 |
|--------|------|
| 10 | 安全目標違反（人身傷害の可能性） |
| 8 | フェールセーフ遷移必要（機能喪失） |
| 6 | 機能低下（安全目標は維持） |
| 4 | 軽微な機能影響 |
| 2 | 影響なし |

**発生頻度（O: Occurrence）**: 1〜10

| レベル | 説明 |
|--------|------|
| 8〜10 | 高（設計寿命内に発生が予想される） |
| 4〜7 | 中（発生の可能性あり） |
| 1〜3 | 低（稀にしか発生しない） |

**検出性（D: Detection）**: 1〜10

| レベル | 説明 |
|--------|------|
| 9〜10 | 検出不可能 |
| 5〜8 | 間接的な検出手段あり |
| 1〜4 | 直接的な診断で検出可能 |

## §2. 故障モード影響解析

### FM-001: ADC固着LOW（電流チャネル）

| 項目 | 内容 |
|------|------|
| FM-ID | FM-001 |
| 故障モード | ADC電流チャネル（ANI0）の出力値がLOW（0）に固着 |
| 局所的影響 | 電流値が常時0mAと計測される |
| システム影響 | 過電流状態（8A超過）が検出されない → SG-002違反（モータ巻線過熱・発火リスク） |
| 関連SG | SG-002 |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 3 |
| 検出方法 | DR-003: ADCプラウジビリティチェック（ADC値が一定回数連続同一値の場合にスタック故障検出） |
| 検出性 (D) | 3 |
| RPN | 90 |
| 対策 | ADCプラウジビリティチェック（SensorProc, TC-001b検証済み）。連続同一値をSTUCK_LIMIT回検出時にfaultフラグSET→DEM経由でフェールセーフ遷移 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（診断メカニズムにより故障検出・安全遷移が保証される） |

### FM-002: ADC固着HIGH（電流チャネル）

| 項目 | 内容 |
|------|------|
| FM-ID | FM-002 |
| 故障モード | ADC電流チャネル（ANI0）の出力値がHIGH（1023）に固着 |
| 局所的影響 | 電流値が常時最大値（約10A）と計測される |
| システム影響 | 誤過電流検出によりモータが不要停止 → 可用性低下（安全側故障） |
| 関連SG | SG-002（安全側） |
| 重大度 (S) | 4 |
| 発生頻度 (O) | 3 |
| 検出方法 | DR-003: ADCプラウジビリティチェック（ADC値が一定回数連続同一値の場合にスタック故障検出） |
| 検出性 (D) | 3 |
| RPN | 36 |
| 対策 | ADCプラウジビリティチェック（SensorProc, TC-001b検証済み）。安全側故障であるため即座に危険とはならないが、スタック検出によりDEM経由で故障報告を実施 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（安全側故障であり人身危害リスクなし。診断により故障通知実施） |

### FM-003: ホールセンサ故障

| 項目 | 内容 |
|------|------|
| FM-ID | FM-003 |
| 故障モード | ホールセンサ（IF-02/03/04）の信号固着または断線によりRPM計測不能 |
| 局所的影響 | 回転数が計測不能（0rpm固定または異常値） |
| システム影響 | 過速度状態（1,500rpm超過）が検出されない → SG-001違反（脱水槽破損・飛散リスク） |
| 関連SG | SG-001 |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 4 |
| 検出方法 | RPM周期監視（期待周期からの逸脱検出）+ WDTフォールバック（WDT TO 100msでMCUリセット→安全状態） |
| 検出性 (D) | 4 |
| RPN | 160 |
| 対策 | 1. RPM周期監視: モータ動作中にホール信号が一定期間無変化の場合、センサ故障と判定しフェールセーフ遷移<br>2. WDTフォールバック: ホールセンサ故障によりモータ制御が異常動作した場合、WDT TO 100msでMCUリセット→HW保証によるゲート遮断 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（RPM監視+WDTの2重防護。WDTリセット時のHW保証設計によりゲート遮断保証） |

### FM-004: MCUクロックドリフト

| 項目 | 内容 |
|------|------|
| FM-ID | FM-004 |
| 故障モード | メインクロック（32MHz）の周波数が±4%を超えてドリフト |
| 局所的影響 | タイマ周期・ADCサンプリングレート・PWM周波数の逸脱 |
| システム影響 | 安全メカニズムのタイミング違反 → SG-001〜SG-007の応答時間保証が喪失 |
| 関連SG | 全SG（タイミング基盤） |
| 重大度 (S) | 8 |
| 発生頻度 (O) | 3 |
| 検出方法 | クロック監視（MEM_ClockCheck, TSR-011）: 外付け水晶（32.768kHz）をサブクロック基準とし、TAUタイマによりメインクロック周期を計測。±4%逸脱を検出 |
| 検出性 (D) | 2 |
| RPN | 48 |
| 対策 | クロック監視（MEM_ClockCheck）: 外付け水晶基準（±20ppm精度）でメインクロック周波数を監視。異常検出時にDEM経由でフェールセーフ遷移。TC-014で検証済み |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（高精度水晶基準による検出、IEC 60730 Annex H Table H.4準拠） |

### FM-005: RAMビット反転（安全変数）

| 項目 | 内容 |
|------|------|
| FM-ID | FM-005 |
| 故障モード | RAM上の安全フラグ変数にビット反転が発生（EMI/放射線等による） |
| 局所的影響 | 安全フラグが不正値に変化（例: 過電流フラグが強制クリア） |
| システム影響 | 安全監視機能の喪失 → 対応するSGの違反可能性 |
| 関連SG | 全SG（安全変数依存） |
| 重大度 (S) | 8 |
| 発生頻度 (O) | 3 |
| 検出方法 | RAM MarchCランタイムテスト（TSR-009）: 256B×22ブロック、全周期2.2sで全RAM領域を診断 |
| 検出性 (D) | 2 |
| RPN | 48 |
| 対策 | 1. MarchCランタイムテスト: 1ブロック/制御周期で分割実行、2.2sで全RAM完了<br>2. 安全変数CRC保護: 安全関連変数にCRC付きデータ構造を使用（SafetyMon, TC-006検証済み）<br>3. 起動時フルテスト: MEM_StartupTestで全ブロック一括検査（TC-012b検証済み） |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（MarchC + CRC保護の2重防護。IEC 60730 Annex H Table H.2準拠） |

### FM-006: ROM内容破壊

| 項目 | 内容 |
|------|------|
| FM-ID | FM-006 |
| 故障モード | Flash ROM上のプログラムコードまたは定数データが破壊 |
| 局所的影響 | コードロジック異常、定数値不正 |
| システム影響 | 制御ロジック異常 → 予期しないモータ動作、安全機能の喪失 |
| 関連SG | 全SG（コード基盤） |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 2 |
| 検出方法 | ROM CRC-32検査（TSR-010）: 全64KB Flashに対するCRC-32チェック |
| 検出性 (D) | 2 |
| RPN | 40 |
| 対策 | 1. 起動時: 全Flash CRC-32検査（MEM_StartupTest, TC-012b検証済み）<br>2. ランタイム: 分割CRC-32検査（MEM_RunDiag, TC-012c検証済み）<br>3. CRC不一致時: DEM経由でフェールセーフ遷移 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（CRC-32は高DC（99%）。IEC 60730 Annex H Table H.3準拠） |

### FM-007: 蓋センサ配線断線

| 項目 | 内容 |
|------|------|
| FM-ID | FM-007 |
| 故障モード | 蓋センサ（IF-07, P43）の配線が断線 |
| 局所的影響 | GPIO入力がプルアップにより常時HIGH（蓋閉状態と同等） |
| システム影響 | 蓋開放が検出されない → SG-003違反（回転ドラムへの手巻き込みリスク） |
| 関連SG | SG-003 |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 4 |
| 検出方法 | デバウンス処理によるフェイルセーフオープン設計: 蓋センサはプルアップ構成（断線時=蓋開放検出→安全側故障） |
| 検出性 (D) | 2 |
| RPN | 80 |
| 対策 | フェイルセーフオープン設計: 蓋センサは「断線時=開放状態検出」のプルアップ構成を採用。断線時はGPIO入力がHIGH→デバウンスにより蓋開放と判定→モータ停止。**注記**: プルアップ構成のため、断線は蓋開放として検出される（安全側故障）。蓋閉状態の短絡故障は別途検討が必要（PH-07） |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（フェイルセーフオープン設計により断線は安全側故障として処理） |

### FM-008: PWM出力固着ON

| 項目 | 内容 |
|------|------|
| FM-ID | FM-008 |
| 故障モード | PWM出力（TAU0 Ch0-5）がHIGH固着しモータが停止不能 |
| 局所的影響 | PWMデューティ制御不能（100%固定） |
| システム影響 | モータが停止できない → SG-001違反（過速度）、SG-002違反（過電流） |
| 関連SG | SG-001, SG-002 |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 3 |
| 検出方法 | HWコンパレータ（経路3、MCU完全非依存）+ WDTリセット（経路2、GPIO非依存） |
| 検出性 (D) | 3 |
| RPN | 90 |
| 対策 | 1. HWコンパレータ（経路3）: 電流がHW閾値を超過した場合、MCU非依存でゲートドライバFault信号を発行→インバータゲート遮断<br>2. WDTリセット（経路2）: MCU異常によるPWM固着の場合、WDT TO 100ms→MCUリセット→GPIOデフォルトHi-Z→ゲートドライバEnable=LOW→ゲート遮断<br>3. 独立遮断経路によりSW制御パス故障時もHW単独で安全状態達成 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（3系統独立遮断経路による多重防護。TC-411で検証予定） |

### FM-009: MCU完全故障（CPUハング）

| 項目 | 内容 |
|------|------|
| FM-ID | FM-009 |
| 故障モード | MCU（RL78/G14）のCPUが完全にハング（無限ループ、プログラムカウンタ固着等） |
| 局所的影響 | 全SW機能停止（安全監視、モータ制御、自己診断の全て停止） |
| システム影響 | 安全機能完全喪失 → 全SG違反の可能性 |
| 関連SG | 全SG |
| 重大度 (S) | 10 |
| 発生頻度 (O) | 3 |
| 検出方法 | WDTリセット（100ms タイムアウト）+ HWコンパレータ（MCU完全非依存） |
| 検出性 (D) | 2 |
| RPN | 60 |
| 対策 | 1. WDTリセット（経路2）: CPUハング→WDTキック停止→100ms後にWDTリセット→GPIOデフォルトHi-Z→ゲート遮断（HW保証）<br>2. HWコンパレータ（経路3）: 過電流発生時はMCU非依存でゲートFault発行→ゲート遮断<br>3. WDTは独立タイマ源を使用し、メインクロック障害時もタイムアウト動作を保証 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（WDT + HWコンパレータの独立2経路防護。TC-404, TC-411で検証予定） |

### FM-010: 電圧レギュレータ故障

| 項目 | 内容 |
|------|------|
| FM-ID | FM-010 |
| 故障モード | 電圧レギュレータ故障によりVDDが4.5V〜5.5V範囲外に逸脱 |
| 局所的影響 | MCUが仕様外電圧で動作（誤動作、ラッチアップの可能性） |
| システム影響 | MCU誤動作による予期しない出力 → 全SG違反の可能性 |
| 関連SG | SG-006（直接）、全SG（間接） |
| 重大度 (S) | 8 |
| 発生頻度 (O) | 3 |
| 検出方法 | 電圧監視（TSR-013, ADC 10Hz）+ ブラウンアウトリセット（MCU内蔵） |
| 検出性 (D) | 3 |
| RPN | 72 |
| 対策 | 1. 電圧監視（FSR-009/010）: ADCにより10HzでVDDを監視。4.5V未満または5.5V超過でモータ停止（TC-405検証予定）<br>2. ブラウンアウトリセット: MCU内蔵BOD回路により極低電圧時にリセット→GPIOデフォルトHi-Z→ゲート遮断（HW保証）<br>3. デュアル閾値監視: 低電圧（4.5V）と高電圧（5.5V）の双方向監視 |
| 対策状態 | **対策実装済み** |
| 残余リスク | **残余リスク受容**（SW電圧監視 + HWブラウンアウトリセットの2重防護） |
