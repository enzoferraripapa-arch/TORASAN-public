## 3. 各ユニット詳細設計

### 3.1 SA-001 APP_MOT（モータ制御）

#### UT-001: AppMot_Init
- **機能**: PWM初期化、安全変数初期化、制御パラメータゼロクリア
- **呼出元**: main.c Phase 4

#### UT-002: AppMot_Cyclic10ms
- **機能**: RPM計測→速度制御→6ステップ転流
- **周期**: 10ms
- **アルゴリズム**:

```
1. hall_period = HalTimer_GetHallPeriod()
2. raw_rpm = 2500000 / hall_period  (整数除算)
3. filtered_rpm = 移動平均(3サンプル)
4. SafetyVar_t に格納 (逆コピー保護)
5. rpm_error = target - filtered
6. duty_adjust = error / 2  (P制御)
7. hall_pattern = HalGpio_GetHallPattern()
8. 転流テーブル[hall_pattern] → phase_high/low
9. HalPwm_SetDuty(phase_high, duty)
```

- **SafetyVar_t検証**: 毎サイクルでSAFETYVAR_CHECK実施、不整合時はEmergencyStop
- **6ステップ転流テーブル**:

| Hall (W|V|U) | High Phase | Low Phase |
|-------------|-----------|----------|
| 001 | U | V |
| 010 | W | U |
| 011 | W | V |
| 100 | V | W |
| 101 | U | W |
| 110 | V | U |

#### UT-003: AppMot_EmergencyStop
- **機能**: PWM全停止 → リレーOFF → 状態クリア
- **応答時間**: ≤1ms（ソフトウェア処理のみ）
- **呼出元**: AppSaf_TransitionSafe, AppMot_Cyclic10ms(異常時)

---

### 3.2 SA-002 APP_SAF（安全監視）

#### UT-004: AppSaf_Init
- **機能**: 状態=STARTUP_DIAG、デバウンスカウンタゼロ、LPF初期値設定

#### UT-005: AppSaf_Cyclic10ms
- **機能**: 4つの安全チェックを周期実行
- **チェック項目と判定基準**:

| チェック | SR | 閾値 | デバウンス | 検出時間 |
|---------|-----|------|----------|---------|
| 過速度 | SR-002 | >1500rpm | 3回 (30ms) | ≤30ms |
| 過電流 | SR-005 | >8000mA | 2回 (20ms) | ≤20ms |
| 蓋開放 | SR-007 | LID=OPEN | 3回 (30ms) | ≤30ms |
| 電圧異常 | SR-014 | <4500mV or >5500mV | 3回 | ≤300ms (10Hz) |

- **LPF実装**: 整数固定小数点(Q8)
  - `y = (alpha * x + (256 - alpha) * y_prev) / 256`
  - 電流: alpha=51 (≈0.2)、電圧: alpha=26 (≈0.1)

#### UT-006: AppSaf_TransitionSafe
- **機能**: EmergencyStop → 状態=SAFE → LED赤 → DEM報告 → NVMログ
- **応答時間**: Stage1（モータ停止）≤1ms

#### UT-007: AppSaf_GetState
- **機能**: 現在のSystemState_tを返却

---

### 3.3 SA-003 HAL（ハードウェア抽象化）

#### UT-008: HalAdc_*
- **ADC設定**: 10bit、ソフトウェアトリガ、fCLK/64
- **チャネル**: CH0=電流、CH1=電圧、CH2=診断基準
- **変換式**: current_mA = (raw * 10000) / 1023、voltage_mV = (raw * 5000) / 1023

#### UT-009: HalGpio_*
- **入力**: P7.0(蓋センサ)、P3.0-2(ホールU/V/W)
- **出力**: P2.1(リレー)、P2.2(赤LED)、P2.3(緑LED)

#### UT-010: HalPwm_*
- **搬送波**: 20kHz (32MHz / 1600カウント)
- **分解能**: 0.1% (duty 0-1000)
- **AllStop**: 全duty=0 + 出力ディセーブル

#### UT-011: HalTimer_*
- **CH00**: 1msシステムチック (fCLK/32 = 1MHz, 1000カウント)
- **CH01**: ホールセンサキャプチャ (フリーランニング、1us分解能)
- **TickIncrement**: タイマISRから呼出

#### UT-012: HalUart_*
- **設定**: 9600bps、8N1
- **送受信**: ブロッキング方式（本実装）

---

### 3.4 SA-004 WDT_MGR

#### UT-013: WdtMgr_*
- **タイムアウト**: 100ms
- **キック条件**: safety_flag の全3ビット(MOTOR/SAFETY/DIAG)がセットされていること
- **キックパターン**: 0xAC (RL78 WDT)
- **フェイルセーフ**: 条件未達時はキック拒否 → MCUリセット

---

### 3.5 SA-005 DEM

#### UT-014: Dem_*
- **エラー集約**: fault_mask (16bitビットマスク) + last_fault + fault_count
- **ReportError**: ビットセット + カウンタ加算(飽和)
- **HasActiveFault**: mask != 0 で判定

---

### 3.6 SA-006 MEM_MGR

#### UT-015: MemMgr_*
- **NVMエミュレーション**: RAM上の循環バッファ（本番はデータフラッシュ使用）
- **容量**: 16エントリ × 6byte = 96byte
- **WriteLog**: 循環上書き、ReadLog: index 0 = 最新

---

### 3.7 SA-007 DIAG（自己診断）

#### UT-016: Diag_RunStartup (DR-001)
- **シーケンス**: CPU Reg → RAM March C (全22ブロック) → ROM CRC32 (全64KB) → Clock
- **制限時間**: ≤700ms
- **いずれかFAIL**: 即座にAppSaf_TransitionSafe、永久ロック

#### UT-017: Diag_RunCyclic10ms (DR-002)
- **RAM**: 1ブロック/サイクル (256B)、全体2.2s周期
- **ROM**: 1KB/サイクル、全体≤5s周期
- **Clock**: 10サイクルごと (100ms周期)
- **ADC/GPIO**: 2サイクルごとに交互

#### UT-018: 個別診断
- **CpuRegTest**: 4パターン (0x55AA, 0xAA55, 0xFFFF, 0x0000)
- **RamTestBlock**: March C 6パス (W0↑, R0W1↑, R1W0↑, R0↓) on 256B
- **RomCrcStep**: CRC-32 (polynomial 0x04C11DB7), 1KB/step
- **ClockTest**: メイン/サブクロック比較、±4%許容

#### UT-019: 周辺診断
- **AdcTest**: 診断基準チャネル mid-scale ±10%、5回連続異常で確定
- **GpioTest**: リレーピン書込み→読戻し検証

---

### 3.8 SA-008 COM

#### UT-020: Com_*
- **プロトコル**: [STX][LEN][CMD][DATA...][CRC8(XOR)]
- **StatusFrame**: 8byte (STX + LEN + CMD + STATE + RPM_H + RPM_L + FAULT + CRC8)

## 4. 安全設計根拠

### 4.1 IEC 60730 Annex H 適合性

| Annex H 要件 | 対応UT | 実装方式 | DC |
|-------------|--------|---------|-----|
| H.2.1 WDT | UT-013 | 条件付きキック、50ms周期、100msタイムアウト | 99% |
| H.2.2 パラメータ範囲チェック | UT-005 | 過速度/過電流/電圧ウィンドウ+デバウンス | 90% |
| H.3.1 CPUレジスタ | UT-018 | 4パターンテスト | 95% |
| H.3.2 RAM静的テスト | UT-018 | March C (6N+4)、256B×22ブロック | 90% |
| H.3.3 ROMチェック | UT-018 | CRC-32、64KB全域 | 99% |
| H.3.5 クロック監視 | UT-018 | メイン/サブクロック比較 ±4% | 85% |

### 4.2 多層冗長性

```
層1: ソフトウェア監視 (APP_SAF)
  ├── 過速度検出 → EmergencyStop
  ├── 過電流検出 → EmergencyStop + リレーOFF
  ├── 電圧異常   → EmergencyStop
  └── 蓋開放     → EmergencyStop

層2: 自己診断 (DIAG)
  ├── 起動時全項目テスト → 失敗時永久ロック
  └── ランタイム増分テスト → 失敗時SafeState

層3: ハードウェア保護 (WDT_MGR)
  └── WDTタイムアウト → MCUリセット → 起動時診断から再開
```

### 4.3 SafetyVar_t 適用変数

| 変数 | モジュール | 保護理由 |
|------|---------|---------|
| s_motor_rpm | APP_MOT | 過速度判定の入力値、改竄防止 |
