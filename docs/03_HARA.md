# ハザード分析およびリスク評価（HARA）
## 洗濯機BLDCモータ制御ECU

文書番号: WMC-HARA-001 / 版数: 2.1 / 作成日: 2026-03-02

**適用規格**: IEC 60730-1:2023 Class B (製品安全分類) + IEC 60335-2-7 (製品安全)
**プロセス基盤**: ISO 26262:2018 Part 3 §6 の手法をリスク分析フレームワークとして活用

## 1. 使用シナリオ・動作モード定義

本HARAは、PH-02 アイテム定義書（v2.0）で定義された運用シナリオ OP-01〜OP-05 に基づき、各動作モードにおけるハザードを体系的に同定する。

### 1.1 動作モード一覧

| OP-ID | モード名 | 回転数 | 回転方向 | 電流上限 | 蓋ロック | 危険度 |
|-------|---------|-------|---------|---------|---------|-------|
| OP-01 | 洗い | 〜400rpm | 正逆交互 | 6A | ロック | 中 |
| OP-02 | すすぎ | 〜400rpm | 正逆交互 | 6A | ロック | 中 |
| OP-03 | 脱水 | 〜1200rpm | 一方向 | 6A | ロック | **高** |
| OP-04 | 待機 | 0rpm | — | 0A | 解除 | 低 |
| OP-05 | 異常停止 | 0rpm | — | 0A | ロック維持 | 低 |

### 1.2 危険領域定義

- **脱水槽（ドラム）**: OP-03 で定格1200rpm、エネルギー最大
- **電気系統**: 電源供給（AC100V）、インバータ駆動、モータ巻線
- **ユーザー接触領域**: 蓋開口部、操作パネル、筐体表面
- **制御系統**: MCU（RL78/G14）、センサ入力、PWM出力

## 2. ハザード同定リスト

### 2.1 同定手法

以下のハザードは、PH-02 アイテム定義書の HZ-01〜HZ-06 を基礎とし、各運用モード（OP-01〜OP-05）について以下の故障カテゴリを体系的に分析して同定した:

- 機能喪失（制御不能、停止不能）
- 誤動作（意図しない動作、過剰動作）
- センサ故障（検出不能、誤検出）
- 電気異常（過電圧/低電圧、過電流、漏電）
- MCU内部故障（メモリ破損、PC暴走、クロック異常）

### 2.2 ハザード一覧表

| HZ-ID | ハザード | 関連OP | 危険事象 | 発生メカニズム |
|-------|---------|--------|---------|--------------|
| HZ-01 | モータ過速度 | OP-03 | 脱水槽の異常振動・破損・飛散による負傷 | PWM制御喪失により定格1200rpmを大幅超過（1500rpm以上） |
| HZ-02 | モータロック（過電流） | OP-01/02/03 | 過電流（8A超過）による発熱→発火・焼損 | 機械的ロックまたは制御異常による通電継続 |
| HZ-03 | 蓋開放中のモータ動作 | OP-01/02/03 | 回転する脱水槽への手の巻き込み | 蓋センサ故障（接触不良/断線）による開放未検知 |
| HZ-04 | 漏電・絶縁劣化 | 全モード | 感電による生命危険 | 水分侵入、長期使用による絶縁劣化、モータ巻線損傷 |
| HZ-05 | MCU異常・制御暴走 | 全モード | 予期しないモータ動作、停止不能 | MCU内部エラー（メモリビット反転、PC異常、スタック破損、電源断後の不正再起動） |
| HZ-06 | 電圧異常 | 全モード | 回路損傷、MOSFET破壊、制御喪失 | 電源異常（過電圧/低電圧: VDD 4.5V〜5.5V逸脱）、雷サージ、電源瞬断 |

**注**: HZ-05 は PH-02 アイテム定義の「電源断時の不正再起動」を包含する上位ハザード。HZ-06 は「センサ故障による異常制御」の電圧起因部分を包含。センサ故障は HZ-01〜03 の発生メカニズムとしても考慮。

## 3. リスク評価

### 3.1 評価手法

本HARAでは、以下の2段構えでリスク評価を行う:

1. **IEC 60730 安全分類**（製品規制準拠）: IEC 60730-1 に基づく Class A/B/C 分類で製品安全等級を決定
2. **S×E×C 補足リスク分析**: ISO 26262 Part 3 の評価スケールを体系的リスク分析ツールとして活用し、安全目標の優先順位付けとFTTI分析の根拠とする

**重要**: ISO 26262 の ASIL 判定（S×E×C → ASIL A/B/C/D）は車載製品固有の分類であり、本製品（家庭用洗濯機）には適用しない。ISO 26262 はプロセス手法（V-model、SPICE、FMEA、FTTI等）のベストプラクティスとしてのみ採用している。

### 3.2 IEC 60730 安全分類

#### 分類基準（IEC 60730-1 Annex H）

| 分類 | 定義 | ソフトウェア要件 | 対象例 |
|------|------|---------------|--------|
| Class A | 安全に関係しない制御機能 | 特別要件なし | タイマー、表示機能 |
| **Class B** | **機器の安全でない動作を防止する制御機能** | **自己診断（Annex H）必須** | **過速度保護、過電流遮断、蓋インターロック、MCU自己診断** |
| Class C | 特殊危険（ガス・爆発等）を防止する制御機能 | 冗長設計必須 | ガス燃焼制御、ボイラー制御 |

#### 本製品の分類根拠

- **対象機器**: 家庭用洗濯機（IEC 60335-2-7 適用製品）
- **安全制御機能**: モータ速度制限、過電流保護、蓋インターロック、MCU自己診断、電圧監視
- **分類判定**: これらは「機器の安全でない動作を防止する」制御機能 → **IEC 60730 Class B** を適用
- **Class C 非該当**: 本製品はガス・爆発等の特殊危険を扱わないため Class C には該当しない

### 3.3 S×E×C 補足リスク分析

ISO 26262 Part 3 の S（重大度）×E（暴露頻度）×C（制御可能性）スケールを、安全目標の優先順位付けのための補足的リスク分析ツールとして使用する。

#### Severity（S: 重大度）

| レベル | 説明 | 例 |
|--------|------|-----|
| S3 | 致命的・生命を脅かす | 生命喪失、重傷 |
| S2 | 重大な傷害 | 骨折、深い切傷、火傷 |
| S1 | 軽微な傷害 | 軽い切傷、軽度火傷 |
| S0 | 傷害なし | 不快感のみ |

#### Exposure（E: 暴露頻度）

| レベル | 説明 | 頻度 |
|--------|------|------|
| E4 | 高確率で暴露 | ほぼ常時（使用時間の大半） |
| E3 | 中確率で暴露 | 毎回の運転で発生し得る |
| E2 | 低確率で暴露 | 時々発生 |
| E1 | 稀に暴露 | 異常時のみ |

#### Controllability（C: 制御可能性）

| レベル | 説明 | 状況 |
|--------|------|------|
| C3 | 制御困難 | 一般ユーザーでは回避不可能 |
| C2 | 制御可能性低 | 対応可能だが困難 |
| C1 | 容易に制御可能 | ユーザーが通常の注意で対応可能 |

### 3.4 リスク評価マトリクス

| SG-ID | HZ-ID | 危険事象 | 具体的シナリオ | S | E | C | リスク優先度 | 安全分類 | 安全目標 |
|-------|-------|---------|------------|---|---|---|-----------|---------|---------|
| SG-001 | HZ-01 | 脱水槽破損飛散 | OP-03 脱水中にPWM制御回路が喪失、回転数が1500rpm超過。遠心力で脱水槽が破損し金属片飛散 | S2 | E3 | C3 | **高** | Class B | モータ回転数が1500rpm（`product_spec.motor.overspeed_rpm`）を超過した場合、WDTタイムアウト（100ms）以内にインバータを停止すること |
| SG-002 | HZ-02 | 過電流発火 | OP-01/02/03 でモータが機械的にロック、電流制限故障により8A超過で通電継続。モータ・ケーブルが過熱し発火 | S2 | E3 | C3 | **高** | Class B | モータ電流が8A（`product_spec.motor.max_a`）を超過した場合、50ms以内に通電を遮断すること |
| SG-003 | HZ-03 | 手の巻き込み | OP-01/02/03 中にユーザーが蓋を開放。蓋センサ故障（接触不良）で未検知、回転する槽に手を巻き込まれる | S2 | E2 | C2 | **中** | Class B | 蓋開放を検知した場合、200ms以内にモータを停止すること。蓋センサの定期診断（診断周期≦制御周期）を実装 |
| SG-004 | HZ-04 | 感電 | 長期使用・水分侵入で絶縁劣化。ユーザーが筐体接触時に漏電による感電 | S3 | E1 | C3 | **高** | Class B | 漏電検出時、即座に電源を遮断すること（HW: 漏電遮断器RCD設置。SW範囲: 絶縁異常の間接監視） |
| SG-005 | HZ-05 | 制御暴走 | MCU内部エラー（RAMビット反転、スタック破損、PC異常アドレス）により制御プログラムが逸脱。モータが無制御で回転継続 | S2 | E2 | C3 | **中** | Class B | MCU異常を検出した場合、100ms（`product_spec.wdt.timeout_ms`）以内にフェールセーフ状態（モータ停止、全出力遮断）へ遷移すること |
| SG-006 | HZ-06 | 電圧異常損傷 | 異常電圧（VDD < 4.5V or > 5.5V）供給によりインバータMOSFETゲート破壊、またはMCU電源不安定で制御喪失 | S1 | E2 | C2 | **低** | Class B | 電源電圧が4.5V〜5.5V（`product_spec.voltage_monitor`）範囲外の場合、モータ動作を禁止しエラー表示を行うこと |

### 3.5 安全分類適用結果

| SG | S×E×C リスク優先度 | 安全分類 | 分類根拠 |
|----|-----------------|---------|---------|
| SG-001 | 高 (S2×E3×C3) | **Class B** | 過速度防止 — 機器の安全でない動作を防止する制御機能 |
| SG-002 | 高 (S2×E3×C3) | **Class B** | 過電流防止 — 機器の安全でない動作を防止する制御機能 |
| SG-003 | 中 (S2×E2×C2) | **Class B** | 蓋インターロック — IEC 60730 / IEC 60335-2-7 必須安全機能 |
| SG-004 | 高 (S3×E1×C3) | **Class B** | 感電防止 — HW対策主体（RCD）、IEC 60335 必須 |
| SG-005 | 中 (S2×E2×C3) | **Class B** | MCU自己診断 — IEC 60730 Annex H Class B 必須要件 |
| SG-006 | 低 (S1×E2×C2) | **Class B** | 電圧監視 — IEC 60730 Annex H Class B 要件（異常電圧での安全動作） |

**判定**: 全安全目標を **IEC 60730 Class B** で統一。IEC 60730 の Class B 自己診断要件（Annex H）が全安全機能に適用されるため、S×E×C リスク優先度に関わらず一律 Class B を適用する。

## 5. セキュリティ脅威分析（TARA 予備評価）

PH-02 アイテム定義書で識別した脅威 TH-01〜TH-03 について、HARAと並行して予備的リスク評価を実施する。

### 5.1 脅威一覧と予備評価

| TH-ID | 脅威 | 攻撃ベクター | 影響を受けるSG | 予備リスク | 対応方針 |
|-------|------|-----------|-------------|----------|---------|
| TH-01 | 不正モータ制御 | ネットワーク経由の不正コマンド | SG-001, SG-002 | **TBD** (ネットワーク有無に依存) | 入力バリデーション、認証、コマンドフィルタリング |
| TH-02 | ファームウェア改竄 | OTA更新機構の悪用、デバッグポート経由 | SG-005 (ROM改竄) | **TBD** | セキュアブート、CRC32検証（SF-07で一部対応）、デバッグポート無効化 |
| TH-03 | データ漏洩 | 使用パターンの個人情報流出 | — (安全性への直接影響なし) | **低** | 通信暗号化（ネットワーク接続時） |

### 5.2 Safety-Security 相互影響

| 脅威 | 安全機能への影響 | 緩和策 |
|------|--------------|-------|
| TH-01 → SG-001 | 不正コマンドで過速度を誘発 | SF-01（過速度検出）が最終防壁として機能 |
| TH-01 → SG-002 | 不正コマンドでモータロック誘発 | SF-02（過電流検出）が最終防壁として機能 |
| TH-02 → SG-005 | 改竄ファームウェアで安全機能を無効化 | SF-07（CRC32）で起動時検出。ランタイム保護は追加検討 |

### 5.3 判定

- ネットワーク接続仕様（TBD-06）の確定前のため、TH-01/TH-02 は **TBD** とする
- TH-03 は安全性への直接影響なし、プライバシー対策として記録
- **既存安全機能（SF-01, SF-02, SF-07）がセキュリティ脅威に対する多層防御の一部として機能**
- 正式な TARA は PH-04（FSC）以降で TBD-06 確定後に実施
