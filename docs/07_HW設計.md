## 3. HW安全機構

IEC 60730 Class B に基づく 3 つの HW 安全機構を定義する。

### 3.1 WDTハードウェアリセット（HW-REQ-004）

| 項目 | 仕様 |
|------|------|
| 使用WDT | RL78/G14 内蔵ウォッチドッグタイマ |
| タイムアウト | 100ms（`product_spec.wdt.timeout_ms`） |
| キック周期 | 50ms（`product_spec.wdt.kick_ms`） |
| キック条件 | メインループ全処理正常完了時のみ |
| タイムアウト動作 | MCU強制リセット → スタートアップ診断 → 安全状態遷移 |
| 設定方法 | オプションバイトにて設定（プログラムで変更不可） |

**設計根拠**: WDTタイムアウト(100ms)はFTTI(100ms)以内。キック周期(50ms)はタイムアウトの50%でジッタマージン確保。オプションバイト設定によりSW暴走時の無効化を防止。

### 3.2 電圧監視（HW-REQ-006）

| 項目 | 仕様 |
|------|------|
| 監視対象 | VDD 5.0V電源ライン |
| 監視方法 | 分圧回路 → ADC ch1 (ANI1) + 電圧監視IC |
| 正常範囲 | 4.5V 〜 5.5V（`product_spec.voltage_monitor`） |
| サンプリング | 10Hz（`product_spec.adc.voltage_hz`） |
| 異常時動作 | 電圧監視IC: MCUリセット出力。SW: 安全状態遷移 |

**分圧回路設計**:

```
VDD (5.0V)
  │
  ├─ R_high (10kΩ)
  │
  ├─── ANI1 入力 → ADC ch1
  │
  ├─ R_low (10kΩ)
  │
  GND

ADC値 = (VDD / 2) / Vref × 1023
      = (5.0 / 2) / 5.0 × 1023
      = 511 (中央値)

下限閾値: (4.5 / 2) / 5.0 × 1023 = 460
上限閾値: (5.5 / 2) / 5.0 × 1023 = 563
```

**電圧監視IC**: 独立した電圧監視ICでVDD低下時にMCUリセットを発行。SW監視とのデュアル監視構成でIEC 60730 Annex H Table H.11.12.7 要求を充足。

### 3.3 過電流保護（HW-REQ-002, HW-REQ-010）

| 項目 | 仕様 |
|------|------|
| 保護対象 | モータ駆動電流（定格6A、最大8A） |
| 1次保護 | ヒューズ/ポリスイッチ（8A定格） |
| 2次保護 | SW過電流検出 → インバータEN=LOW → ゲート遮断 |
| 電流検出 | シャント抵抗(0.01Ω) + 差動アンプ → ADC ch0 (ANI0) |
| サンプリング | 100Hz（`product_spec.adc.current_hz`） |
| 遮断応答 | SW: ≦50ms、HW(ヒューズ): 即時 |

**電流検出回路設計**:

```
インバータ DC Bus
  │
  ├─ シャント抵抗 R_shunt (0.01Ω, 0.1%, 2W)
  │
  ├──┬── 差動アンプ (Gain=50)
  │  │   → 出力: 0〜4.0V (0〜8A相当)
  │  │   → ANI0 入力 → ADC ch0
  │  │
  GND

ADC値 = (I × R_shunt × Gain) / Vref × 1023
 8A時 = (8 × 0.01 × 50) / 5.0 × 1023 = 819
 6A時 = (6 × 0.01 × 50) / 5.0 × 1023 = 614
```

**過電流閾値**: ADC値 > 819 (8A相当 = `product_spec.motor.max_a`) で即時遮断。

**G-07-02 検証**: WDT(§3.1) + 電圧監視(§3.2) + 過電流保護(§3.3) = 3機構全て定義済み → PASS

## 4. 回路設計

### 4.1 3相インバータ回路

```
DC Bus (+)
  │
  ├─ ヒューズ/ポリスイッチ (8A)
  │
  ├──┬──────────┬──────────┐
  │  │          │          │
  Q1 (Hi-U)   Q3 (Hi-V)  Q5 (Hi-W)   ← N-ch MOSFET
  │  │          │          │
  ├──┼── U ─────┼── V ─────┼── W ──→ BLDCモータ
  │  │          │          │
  Q2 (Lo-U)   Q4 (Lo-V)  Q6 (Lo-W)   ← N-ch MOSFET
  │  │          │          │
  ├──┴──────────┴──────────┘
  │
  R_shunt (0.01Ω) → 差動アンプ → ANI0
  │
  GND

ゲートドライバIC
  ├─ HIN_U (P10/PWM-U) ──→ Q1 ゲート
  ├─ LIN_U              ──→ Q2 ゲート
  ├─ HIN_V (P11/PWM-V) ──→ Q3 ゲート
  ├─ LIN_V              ──→ Q4 ゲート
  ├─ HIN_W (P12/PWM-W) ──→ Q5 ゲート
  ├─ LIN_W              ──→ Q6 ゲート
  ├─ EN (P10/GPIO)      ──→ イネーブル（LOW=全遮断）
  └─ FAULT#             ──→ MCU INT入力（短絡検出）
```

**デッドタイム**: ゲートドライバIC内蔵（2μs typ.）。Hi/Lo同時ONによるアーム短絡を防止。

### 4.2 ホールセンサ入力回路

```
ホールセンサ出力 (Open Collector)
  │
  ├─ R_pull (4.7kΩ) ─── VDD (5V)
  │
  ├─ R_filter (1kΩ)
  │   │
  │   ├─ C_filter (100pF)
  │   │   │
  │   │   GND
  │   │
  │   └──→ P20/P21/P22 (TAU0-4/5/6 Timer Capture)

ノイズフィルタ時定数: τ = 1kΩ × 100pF = 100ns
（PWM 20kHz = 50μsに対して十分短い）
```

### 4.3 蓋センサ/ロック回路

```
蓋センサ（マイクロスイッチ）
  │
  ├─ R_pull (10kΩ) ─── VDD (5V)
  │
  ├─ R_debounce (10kΩ)
  │   │
  │   ├─ C_debounce (100nF)
  │   │   │
  │   │   GND
  │   │
  │   └──→ P43 (GPIO Input, pull-up)

デバウンス時定数: τ = 10kΩ × 100nF = 1ms
（5ms監視周期に対して十分短く、チャタリング除去に有効）

蓋ロック（ソレノイド駆動）
  P44 (GPIO Output) ──→ トランジスタ ──→ ソレノイド
                         │
                         └─ フライバックダイオード
```

## 9. HW設計制約と今後の課題

### 9.1 設計制約

- **P10兼用**: PWM U相出力とインバータEN信号が同一ピン。遮断時はGPIOモードへ切替が必要。PH-10 SW詳細設計で制御シーケンスを定義する
- **ADC共有**: 電流(ch0)と電圧(ch1)は時分割変換。変換順序と優先度はSW側で管理
- **RAM制約**: 5.5KB RAMでの March C テスト（256B×22ブロック、2.2s全周期）はメインループと並行実行が必要

### 9.2 PH-12（HW統合テスト）への引継事項

- 電流検出回路の実測精度確認（TBD-08: SG-002 電流センサ応答時間）
- 電圧監視ICのリセット閾値実測
- ホールセンサ信号のS/N比確認
- WDTタイムアウト実測（オプションバイト設定の妥当性検証）
- デッドタイム値の実測確認
