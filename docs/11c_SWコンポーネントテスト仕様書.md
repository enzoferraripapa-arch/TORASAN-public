## 1. テスト方針

### 1.1 テスト目的

PH-08 SRS v2.0 で定義された全20件の要求（SR-001〜SR-016 + DR-001〜DR-004）に対し、要求ベースのブラックボックステストを実施し、ソフトウェアが安全要求を満たすことを検証する。

### 1.2 テスト戦略

- **テスト手法**: ブラックボックステスト（要求の受入基準に基づく判定）
- **カバレッジ目標**: 全SR/DR に最低1件のTC（要求カバレッジ100%）
- **テスト分類**: 正常系 / 境界値 / 異常系（故障注入）
- **判定基準**: SRS記載の定量的受入基準を直接使用

### 1.3 テスト環境

- **プラットフォーム**: ホストPC（GCC）+ 将来ターゲット（RL78/G14）
- **フレームワーク**: Unity/CMock
- **計測ツール**: タイマスタブ（応答時間計測）

### 1.4 product_spec パラメータ参照

テストケースで使用する閾値はすべて `project.json.product_spec` から導出:

| パラメータ | 値 | 使用TC |
|----------|-----|-------|
| motor.overspeed_rpm | 1500 rpm | TC-201〜203 |
| motor.max_a / max_ma | 8A / 8000mA | TC-207〜209 |
| voltage_monitor.min_v | 4.5V | TC-228〜230 |
| voltage_monitor.max_v | 5.5V | TC-228〜230 |
| wdt.timeout_ms | 100ms | TC-232〜233 |
| adc.bits | 10bit (0〜1023) | TC-237〜238 |
| ram_test.block_b | 256B | TC-235〜236 |
| clock_tolerance_pct | 4% | TC-224 |
