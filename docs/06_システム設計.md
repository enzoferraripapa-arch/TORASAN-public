### 3.2 信号タイミング仕様

#### PWM信号 (IF-01)
- キャリア周波数: 20kHz（周期50us）
- デューティ比: 0〜100%（0.1%ステップ）
- 駆動方式: 6ステップ転流（ホール位置に基づく）
- 安全停止: デューティ = 0% + EN = LOW で即時遮断

#### ホールセンサ (IF-02〜04)
- 信号: デジタル 5V/GND（3相、120度配置）
- キャプチャ: Timer立上り/立下りエッジで周期計測
- 遷移パターン（正転）: 001→011→010→110→100→101→001
- 無信号検出: 200ms以上遷移なし → 回転停止判定

#### ADC入力 (IF-05, IF-06)
- 分解能: **10bit** / Vref **5.0V** (`product_spec.adc`)
- 分解能値: 5.0V / 1024 ≒ 4.88mV/LSB
- 電流(IF-05): **100Hz** サンプリング（10ms周期）
- 電圧(IF-06): **10Hz** サンプリング（100ms周期）
- 変換時間: ≒4us（RL78/G14）

#### GPIO信号 (IF-07, IF-08, IF-12)
- 信号レベル: CMOS (0/5V)
- IF-07蓋センサ: P43, 内部プルアップ — 蓋閉=LOW / 蓋開=HIGH
  - デバウンス: 5ms以上継続確認（TSR-006定義）
- IF-08蓋ロック: P44出力 — ロック=HIGH / 解除=LOW
- IF-12インバータEN: P10出力 — 有効=HIGH / 遮断=LOW

#### WDT (IF-11)
- タイムアウト: **100ms** (`product_spec.wdt.timeout_ms`)
- キック周期: **50ms** (`product_spec.wdt.kick_ms`)
- キック方式: RL78/G14 WDTレジスタへの書込み
- タイムアウト時: MCU HW自動リセット

#### UART (IF-09, IF-10)
- フォーマット: 8N1（8bit, No Parity, 1 Stop bit）
- ボーレート: 9600bps
- プロトコル: Master(MCU)→Slave(Panel)

## 4. ソフトウェア実行モデル

### 4.1 メインループ構成

```
電源ON
  ↓
[スタートアップ診断] ← DIAG モジュール
  ├─ TSR-008: CPUレジスタテスト ≦1ms
  ├─ TSR-009: RAM March C全域テスト ≦50ms
  ├─ TSR-010: Flash CRC32全域検証 ≦500ms
  ├─ TSR-011: クロック監視初期化
  ├─ TSR-015: WDT初期設定(100ms)
  └─ いずれか異常 → TSR-012 安全状態遷移
  ↓ (全PASS)
[メインループ] 10ms周期
  ├─ 入力取得 (HAL)
  │   ├─ IF-02〜04: ホールセンサ → TSR-001 回転数計測
  │   ├─ IF-05: 電流ADC → TSR-004 過電流判定
  │   ├─ IF-06: 電圧ADC (100ms毎) → TSR-013 電圧判定
  │   └─ IF-07: 蓋センサ → TSR-006 デバウンス
  ├─ 安全判定 (APP_SAF)
  │   ├─ TSR-002: 過速度判定 (1500rpm)
  │   ├─ TSR-004: 過電流判定 (8A)
  │   ├─ TSR-014: 電圧異常時起動禁止
  │   └─ 異常検出 → TSR-003/005/007/012 反応
  ├─ モータ制御 (APP_MOT)
  │   └─ PWM演算・出力更新 (IF-01)
  ├─ ランタイム診断 (DIAG) — 分割実行
  │   ├─ TSR-009: RAM March C (256B/100ms周期)
  │   ├─ TSR-010: Flash CRC32 (4KB/周期)
  │   └─ TSR-011: クロック監視 (100ms毎)
  ├─ 通信処理 (COM)
  │   └─ UART送受信 (非安全)
  └─ WDTキック (WDT_MGR)
      └─ TSR-015: 全処理正常完了時のみキック (50ms周期)
```

### 4.2 タイムバジェット

| タスク | 最大実行時間 | 周期 | 備考 |
|--------|-----------|------|------|
| 入力取得(HAL) | 1ms | 10ms | ADC変換+GPIO読取 |
| 安全判定(APP_SAF) | 1ms | 10ms | 閾値比較+状態遷移 |
| モータ制御(APP_MOT) | 3ms | 10ms | PWM演算+転流 |
| ランタイム診断(DIAG) | 2ms | 10ms | 分割実行 |
| 通信(COM) | 1ms | 10ms | UART処理 |
| WDTキック | <0.1ms | 50ms | レジスタ書込みのみ |
| **合計** | **≦8ms** | **10ms** | **余裕: 2ms (20%)** |

## 5. 安全機構のシステムレベル設計

### 5.1 過速度保護（SG-001 → TSR-001/002/003）

```
[IF-02〜04 ホールセンサ] → HAL(Timer Capture)
  → TSR-001: 周期計測 → 回転数算出
  → TSR-002: 閾値比較 (≧1500rpm?)
  → YES → TSR-003: PWM=0% + EN=LOW (IF-01, IF-12)
  → TSR-012: OP-05安全状態遷移
```
- 検出遅延: ≦10ms / 反応時間: ≦1ms / 合計: ≦11ms（FTTI 500ms に対し十分）

### 5.2 過電流保護（SG-002 → TSR-004/005）

```
[IF-05 電流センサADC] → HAL(ADC ch0, 100Hz)
  → TSR-004: 移動平均N=4 → 閾値比較 (≧8A?)
  → YES → TSR-005: PWM=0% + ゲート遮断 (IF-01, IF-12)
  → TSR-012: OP-05安全状態遷移
```
- 検出遅延: ≦50ms(フィルタ含) / 反応時間: ≦1ms / 合計: ≦51ms（FTTI 200ms に対し十分）
- HWバックアップ: ヒューズ（独立経路）

### 5.3 蓋インターロック（SG-003 → TSR-006/007）

```
[IF-07 蓋センサGPIO] → HAL(GPIO, P43)
  → TSR-006: 5msデバウンス → 蓋開放確定
  → TSR-007: PWM=0% + 蓋ロック維持(IF-08) → OP-05
  → 蓋閉鎖確認 → OP-04(待機)に復帰可
```
- 検出遅延: ≦10ms / 反応時間: ≦1ms / 合計: ≦11ms（FTTI 500ms に対し十分）

### 5.4 MCU自己診断（SG-005 → TSR-008〜011, 015）

```
スタートアップ:
  TSR-008: CPUレジスタ 0xAAAA/0x5555 全ビットR/W (≦1ms)
  TSR-009: RAM March C 全域5632B (≦50ms)
  TSR-010: Flash CRC32 全域64KB (≦500ms)
  TSR-011: クロック監視初期化 (32MHz vs 15kHz)
  → いずれかFAIL → TSR-012 安全状態遷移

ランタイム:
  TSR-009: RAM March C 256B/100ms → 全域2.2秒
  TSR-010: Flash CRC32 4KB/周期 → 全域≦16秒
  TSR-011: クロック比較 100ms周期 (許容±4%)
  TSR-015: WDT 100ms TO / 50ms キック
  → 異常検出 → TSR-012 安全状態遷移
```

### 5.5 電圧監視（SG-006 → TSR-013/014）

```
[IF-06 電圧ADC] → HAL(ADC ch1, 10Hz)
  → TSR-013: 移動平均N=4 → 範囲判定 (4.5V〜5.5V)
  → 範囲外 → TSR-014: motor_inhibit = TRUE
  → TSR-012: OP-05安全状態遷移
```
- HWバックアップ: 電圧監視IC（リセット生成 — 独立経路）

### 5.6 安全状態遷移（共通 → TSR-012）

```
[異常トリガ] → APP_SAF
  1. PWM全相OFF (IF-01 = 0%)
  2. インバータEN = LOW (IF-12)
  3. 全出力無効化
  4. DEM: エラーフラグ + ログ記録 (MEM_MGR経由NVM)
  5. COM: エラーコード送信 (IF-09 UART)
  6. 赤色LED点滅 (IF-10)
  → OP-05: 電源リセットまで保持
  → 復帰: 電源OFF→ON → スタートアップ診断PASS → OP-04
```
