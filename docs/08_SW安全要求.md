## 1. 概要

本文書は、PH-05 TSC v2.0（TSR-001〜015）およびPH-06 システム設計 v2.0（3層アーキテクチャ）に基づき、ソフトウェアレベルの安全要求（SR）および自己診断要求（DR）を定義する。

### 1.1 ID体系

| 種別 | ID形式 | 説明 |
|------|--------|------|
| SR-xxx | ソフトウェア安全要求 | TSRから導出されたSW実装要求 |
| DR-xxx | 診断要求 | IEC 60730 Annex H に基づく自己診断要求 |

### 1.2 要求数サマリ

| 区分 | 件数 | 上位TSR |
|------|------|--------|
| SR（ソフトウェア安全要求） | 16件 | TSR-001〜015 全15件から展開 |
| DR（診断要求） | 4件 | TSR-008〜011, 015 から展開 |
| 合計 | 20件 | — |

## 2. ソフトウェア安全要求（SR-001〜SR-016）

### SR-001: ホールセンサ回転数計測

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-001 ← FSR-001 ← SG-001 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/HAL（Timer Capture） + App/APP_MOT |
| **HW IF** | IF-02(P20), IF-03(P21), IF-04(P22) — Timer Capture エッジトリガ |

**機能要求**:
- ホールセンサ3相入力（U/V/W）のエッジタイミングをTAU0-4/5/6のTimer Captureで取得
- 回転数計算: RPM = (エッジカウント / ポールペア数) × 60 / 計測時間
- 計測周期: 10Hz（100ms周期で平均化）
- 有効範囲: 0〜`product_spec.motor.overspeed_rpm`(1500) rpm

**受入基準**:
- 回転数精度: ±5rpm（定格`product_spec.motor.rated_rpm`(1200)rpm時）
- 信号喪失検出: 500ms以内にエッジなし → 回転停止判定
- 応答遅延: ≦100ms（計測周期1周期分）

**検証方法**: テスト（ホールセンサシミュレータによる回転数入力試験）

### SR-002: 過速度閾値判定

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-002 ← FSR-001 ← SG-001 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | App/APP_SAF |

**機能要求**:
- SR-001で計測したRPMを閾値`product_spec.motor.overspeed_rpm`(1500)rpmと比較
- RPM > 1500 → `overspeed_flag` = 1
- ヒステリシス: 回復閾値 = 1500 × 0.9 = 1350rpm（10%ヒステリシス）
- 判定周期: 10ms（メインループ周期）

**受入基準**:
- 検出遅延: ≦20ms（メインループ2周期以内）
- 誤検出防止: 2回連続超過で確定（デバウンス）
- ヒステリシス動作: 1500rpm超過→検出、1350rpm以下→回復

**検証方法**: テスト（閾値境界値試験 + ヒステリシス動作確認）

### SR-003: 過速度時緊急停止

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-003 ← FSR-002 ← SG-001 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | App/APP_SAF + BSW/HAL |
| **HW IF** | IF-01(P10-P12 PWM), IF-12(P10 EN) |
| **関連HW-REQ** | HW-REQ-001 |

**機能要求**:

**受入基準**:
- 応答時間: SR-002検出から PWM停止まで ≦100ms（`product_spec.wdt.timeout_ms`）
- PWM出力: 停止後 U/V/W 全て LOW（デューティ0%）
- EN信号: LOW確認

**検証方法**: テスト（過速度シミュレーション + 応答時間計測）

### SR-004: モータ電流計測

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-004 ← FSR-003 ← SG-002 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/HAL（ADC） + App/APP_SAF |
| **HW IF** | IF-05(ANI0) — ADC ch0 |
| **関連HW-REQ** | HW-REQ-005 |

**機能要求**:
- ADC ch0（ANI0）サンプリング: `product_spec.adc.current_hz`(100)Hz = 10ms周期
- 分解能: `product_spec.adc.bits`(10)bit、Vref = `product_spec.adc.vref_v`(5.0)V
- 電流換算: I[A] = ADC値 × Vref / 1023 / (R_shunt × Gain) = ADC値 × 5.0 / 1023 / 0.5
- 有効範囲: 0〜`product_spec.motor.max_ma`(8000)mA
- ノイズ除去: 1次IIRフィルタ（カットオフ50Hz）

**受入基準**:
- 精度: ±0.2A（定格`product_spec.motor.rated_a`(6)A時）
- サンプリング周期: 10ms ± 1ms
- フィルタ応答: ステップ入力に対し50ms以内で90%到達

**検証方法**: テスト（既知電流源による精度校正試験）

### SR-010: RAM自己テスト（March C）

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-009 ← FSR-007 ← SG-005 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/DIAG |
| **IEC 60730参照** | Annex H Table H.11.2（RAM） |

**機能要求**:

**起動時テスト（DR-001）**:
- 対象: SRAM全領域 `product_spec.mcu.ram_b`(5632)バイト
- アルゴリズム: March C — {↑(w0); ↑(r0,w1); ↑(r1,w0); ↓(r0,w1); ↓(r1,w0); ↑(r0)}

- ブロックサイズ: `product_spec.ram_test.block_b`(256)バイト

- 全周期: `product_spec.ram_test.full_cycle_s`(2.2)秒

**受入基準**:
- 起動時: 全5632B PASS → `status_ram` = OK
- ランタイム: 各ブロック PASS → 正常継続
- FAIL: エラーコード 0x02 → SR-013（安全状態遷移）
- ランタイム全周期: ≦2.2秒

### SR-013: 安全状態遷移制御

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-012 ← FSR-008 ← SG-005 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | App/APP_SAF + BSW/DEM + BSW/MEM_MGR |

**機能要求**:
- トリガ: SR-009〜012 FAIL、SR-002/005異常検出、SR-014/015電圧異常
- 遷移シーケンス:
  1. **即時**(Stage 1): PWM全相停止 + `inverter_stop_flag` = 1
  2. **同時**(Stage 2): `system_safe_mode` = 1 → 割込み制限 → タイマ停止
  3. **次周期**(Stage 3): UI通知（エラーコード表示）→ COMモジュール経由
  4. **保持**(Stage 4): WDTタイムアウト待ち → MCUリセット → 起動診断
- エラーログ: {エラーコード, 発生時刻} をRAMログ領域に記録
- 復帰: 電源オフ→オンのみ（ソフトリセットによる復帰禁止）

**受入基準**:
- Stage 1応答: トリガから PWM停止まで ≦1ms（割込み内処理）
- ログ記録: エラーコード（0x01〜0x05）と発生時刻
- 復帰制限: `system_safe_mode` == 1 の状態でモータ起動コマンド無視

**検証方法**: テスト（各トリガ条件での遷移動作確認 + 復帰制限確認）

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-015 ← FSR-007/008 ← SG-005 |
| **安全分類** | IEC 60730 Class B |

**機能要求**:
- RL78/G14内蔵WDT使用

- キック周期: `product_spec.wdt.kick_ms`(50)ms以下
- キック条件:
  1. ADC入力取得完了

**検証方法**: テスト（キック条件無効化によるタイムアウト確認）

### DR-001: 起動時自己診断シーケンス

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-008〜011 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/DIAG |
| **IEC 60730参照** | Annex H §H.11（電源投入時テスト） |

**機能要求**:
- 電源投入後、以下を順次実行:
  1. SR-009: CPUレジスタテスト → PASS/FAIL
  2. SR-010: RAM March C（全5632B） → PASS/FAIL
  3. SR-011: Flash ROM CRC32（全64KB） → PASS/FAIL
  4. SR-012: クロック初期確認 → 正常/異常
- 判定: 全PASS → OP-04（STANDBY）→ 通常運転許可
- いずれかFAIL → SR-013 → OP-05（安全状態）→ 電源リセット待ち

**受入基準**:
- 全テスト完了時間: ≦700ms（CPU 10ms + RAM 500ms + ROM 100ms + CLK 100ms）
- DC目標: 95%（起動時全数テスト）
- FAIL時: モータ起動禁止を保証

**検証方法**: テスト（各診断のPASS/FAIL組合せによるシーケンス確認）

### DR-003: ADC診断

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-004, 013 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/HAL + BSW/DIAG |
| **IEC 60730参照** | Annex H Table H.11.5（A/D変換器） |

**機能要求**:
- 各チャネル（ch0:電流, ch1:電圧）の変換後チェック:
  - **0固着検出**: 5回連続 ADC値 = 0 → FAIL
  - **最大固着検出**: 5回連続 ADC値 = 0x3FF(1023) → FAIL
  - **範囲逸脱**: 物理的にあり得ない値の継続検出
- FAIL → `adc_error_flag` = 1 → SR-013

**受入基準**:
- 検出条件: 5回連続異常で確定（誤検出防止）
- 検出遅延: ch0(100Hz) → 50ms、ch1(10Hz) → 500ms
- DC目標: 60%（静的故障のサブセット検出）

**検証方法**: テスト（ADC入力固定による固着検出確認）

### DR-004: GPIO診断

| 項目 | 内容 |
|------|------|
| **上位TSR** | TSR-006 |
| **安全分類** | IEC 60730 Class B |
| **割当モジュール** | BSW/HAL + BSW/DIAG |
| **IEC 60730参照** | Annex H Table H.11.6（I/Oポート） |

**機能要求**:
- 安全関連GPIO（蓋センサ P43）の動的テスト:
  - **プルアップ確認**: 入力モード（内部PU有効）→ 読出し → HIGH期待
  - **プルダウン確認**: 出力LOWモード切替 → 読出し → LOW期待
  - 即座に入力モードに復帰
- テスト周期: 1000ms
- テスト中の蓋センサ読取り一時停止（5ms未満）

**受入基準**:
- 断線検出: PU有効なのにLOW → `gpio_error_flag` = 1
- GND短絡検出: PD状態でHIGH → `gpio_error_flag` = 1
- DC目標: 60%
- テスト所要時間: ≦5ms（蓋監視への影響最小化）

**検証方法**: テスト（GPIO断線/短絡シミュレーションによる検出確認）
