# SWアーキテクチャ設計書
## 洗濯機BLDCモータ制御ECU

文書番号: WMC-SWA-001 / 版数: 2.0 / 作成日: 2026-03-02

**適用規格**: IEC 60730-1:2023 Class B (製品安全分類) + IEC 60335-2-7 (製品安全)
**プロセス基盤**: ISO 26262:2018 Part 6 §7 の手法を活用 / **SPICE**: SWE.2

## 1. SWアーキテクチャ概要

### 1.1 設計方針

本SWアーキテクチャは、PH-06 システム設計書の3層アーキテクチャ（App/BSW/HW）をSWコンポーネントに展開し、PH-08 SRS（SR-001〜016 + DR-001〜004 = 20件）の全要求を実現する。

- **アーキテクチャ**: スーパーループ（非RTOS）— 10ms周期メインループ
- **安全分離**: モータ制御（APP_MOT）と安全監視（APP_SAF）を明確に分離
- **HW抽象化**: HAL層が全HWアクセスを抽象化し、移植性を確保
- **自己診断**: DIAG モジュールがIEC 60730 Annex H準拠の診断を統括

### 4.2 安全関連データの保護方式

```c
/* 逆コピー保護（安全変数用） */
typedef struct {
    uint16_t value;     /* 実値 */
    uint16_t inverted;  /* ビット反転値 (~value) */
} SafetyVar_t;

/* 使用例: 回転数共有 */
static SafetyVar_t g_motor_rpm;

static inline bool SafetyVar_Check(const SafetyVar_t* sv) {
    return (sv->value == (uint16_t)(~sv->inverted));
}

/* エラーログ構造体 */
typedef struct {
    uint32_t timestamp_ms;  /* 発生時刻(ms) */
    uint8_t  error_code;    /* 0x01〜0x05 */
    uint16_t raw_value;     /* ADC生値等の補助情報 */
    uint8_t  reserved;      /* パディング(4バイト境界) */
} LogEntry_t;
```

### 4.3 割込みとメインループ間のデータフロー

```
割込みコンテキスト (非安全判定):
  ├─ Timer Capture ISR → g_hall_period[3] 更新（DI/EI保護）
  ├─ ADC完了 ISR       → g_adc_raw[2] 更新（DI/EI保護）
  └─ フラグ設定のみ。安全判定は一切行わない。

メインループコンテキスト (安全判定):
  ├─ 割込みフラグ確認 → DI/EI でraw値コピー → 安全判定実行
  └─ 状態の一貫性を保証（割込みによる途中更新を防止）
```

## 5. 動的振る舞い — スーパーループ実行モデル

### 5.1 起動シーケンス

```
電源ON / WDTリセット
    │
    ▼
[HW初期化] — HAL_Init(): ポート/クロック/ADC/Timer/WDT/UART
    │
    ▼
[起動時自己診断] — Diag_RunStartup() (DR-001)
    ├─ SR-009: CPUレジスタテスト     ≦10ms
    ├─ SR-010: RAM March C (全5632B) ≦500ms
    ├─ SR-011: ROM CRC32 (全64KB)   ≦100ms
    ├─ SR-012: クロック初期確認       ≦100ms
    │
    ├─ 全PASS → WDT初回キック → メインループ開始
    └─ いずれかFAIL → SR-013: 安全状態遷移 → OP-05
```

**起動時間**: ≦700ms（DR-001 受入基準準拠）

### 5.2 メインループ（10ms周期）

```
[メインループ] ── 10ms周期（タイマ割込みでティック管理）
    │
    ├─[1] 入力取得 (HAL) ≦1ms
    │   ├─ HalTimer_GetHallPeriod()   → RPM算出 (SR-001)
    │   ├─ HalAdc_GetCurrent()        → 電流取得 (SR-004)
    │   ├─ HalAdc_GetVoltage()        → 電圧取得 (100ms毎) (SR-014)
    │   └─ HalGpio_GetLidState()      → 蓋状態取得 (SR-007)
    │
    ├─[2] 安全判定 (APP_SAF) ≦1ms
    │   ├─ SR-002: 過速度判定 (>1500rpm? 2回連続)
    │   ├─ SR-005: 過電流判定 (>8A? 2回連続)
    │   ├─ SR-007: 蓋デバウンス (20ms 4回連続)
    │   ├─ SR-014: 電圧範囲判定 (4.5〜5.5V)
    │   └─ 異常検出 → SR-013: Stage1即時停止 (≦1ms)
    │       → SR-003/006/008/015: 各停止シーケンス実行
    │
    ├─[3] モータ制御 (APP_MOT) ≦3ms
    │   ├─ 転流位置決定（ホール3相パターン）
    │   ├─ PWMデューティ演算
    │   └─ HalPwm_SetDuty() 出力更新
    │
    ├─[4] ランタイム診断 (DIAG) ≦2ms — DR-002
    │   ├─ SR-010: RAM March C 1ブロック (256B) → 全22ブロック/2.2s
    │   ├─ SR-011: ROM CRC32 1ブロック (4KB) → 全16ブロック/≦5s
    │   ├─ SR-012: クロック監視 (100ms毎)
    │   ├─ DR-003: ADC固着チェック (5回連続異常で確定)
    │   └─ DR-004: GPIO診断 (1000ms毎)
    │
    ├─[5] 通信 (COM) ≦1ms
    │   └─ UART送受信 (9600bps 8N1)
    │
    └─[6] WDTキック (WDT_MGR) <0.1ms
        └─ SR-016: 5条件全完了時のみ WdtMgr_TryKick()
```

## 7. 安全アーキテクチャ

### 7.1 安全分離の原則

```
┌─────────────────────┐    ┌──────────────────────┐
│ 非安全経路            │    │ 安全経路（Class B）    │
│ ・COM (UART通信)      │    │ ・APP_SAF (安全判定)   │
│ ・運転指令受信        │    │ ・APP_MOT (PWM停止)    │
│                       │    │ ・DIAG (自己診断)      │
│ 非安全経路は安全判定  │    │ ・WDT_MGR (キック制御) │
│ に影響を与えない      │    │ ・DEM (イベント管理)   │
└─────────────────────┘    └──────────────────────┘
         │                           │
         ├───────── HAL ─────────────┤
         │   (HW抽象化: 共用だが     │
         │    安全関連APIは独立管理)   │
         └───────────────────────────┘
```

### 7.2 安全状態遷移シーケンス（SR-013 実装設計）

```
[トリガ] 過速度/過電流/蓋開放/診断FAIL/電圧異常/WDT
    │
    ▼ Stage 1 (≦1ms) — APP_SAF割込みレベル相当
    ├─ HalPwm_AllStop()         → PWM全相OFF
    ├─ HalGpio_SetInverterEn(false) → EN=LOW
    ├─ g_inverter_stop_flag = 1
    │
    ▼ Stage 2 (次メインループ周期)
    ├─ g_system_safe_mode = 1   → 割込み制限
    ├─ Timer停止（PWMキャリア停止）
    │
    ▼ Stage 3 (次メインループ周期)
    ├─ Dem_ReportEvent(code)    → エラーログ記録
    ├─ Com_SendError(code)      → UI通知
    │
    ▼ Stage 4 (保持)
    └─ WDTキック禁止 → 100msでMCUリセット
       → 起動時診断(DR-001) → 全PASS なら OP-04
```

### 7.3 多層防御マッピング

| 安全目標 | 1次防御(SW) | 2次防御(HW) | 3次防御(WDT) |
|---------|-----------|-----------|------------|
| SG-001 過速度 | SR-002→SR-003 (APP_SAF→APP_MOT) | — | SR-016 (100ms TO) |
| SG-002 過電流 | SR-005→SR-006 (APP_SAF) | HW-REQ-010 ヒューズ | SR-016 (100ms TO) |
| SG-003 蓋開放 | SR-007→SR-008 (APP_SAF) | — | SR-016 (100ms TO) |
| SG-005 MCU異常 | DR-001/002 (DIAG) | HW-REQ-003 CLK監視 | SR-016 (100ms TO) |
| SG-006 電圧異常 | SR-014→SR-015 (APP_SAF) | HW-REQ-006 電圧監視IC | SR-016 (100ms TO) |
