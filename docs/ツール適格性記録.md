## 1. 目的と適用範囲

本文書は、ISO 26262 Part 8 §11 および IEC 60730 Annex H に基づき、開発に使用する全ツールの信頼度レベル (TCL) を分類し、適格性を管理する。

### TCL 定義

| TCL | 説明 | 必要な確認 |
|-----|------|----------|
| TCL3 | 高信頼度 — 安全関連出力を生成 | 全出力に対する独立検証が必須 |
| TCL2 | 中信頼度 — オブジェクトコード生成 | ベンダ適格性証明 + 動作検証 |
| TCL1 | 低信頼度 — 補助的利用 | 既知バージョン使用実績確認 |

## 3. ツール個別評価

### 3.1 Claude Code (T-001) — TCL3

**分類**: AI ベース自動生成ツール
**リスク**: 安全関連の設計・コード・テスト成果物を直接生成するため、最高信頼度レベルを適用

**適格性確認方法（方法A: 全成果物レビュー）**:

| 確認項目 | 内容 | 対象フェーズ |
|---------|------|------------|
| A1. 機能安全適合性 | IEC 60730 Annex H / ISO 26262 準拠性確認 | 全フェーズ |
| A2. コード品質 | MISRA C:2012 準拠、静的解析結果確認 | PH-10, PH-11 |
| A3. テスト充足性 | 要件カバレッジ ≧ 90%、MC/DC（IEC 60730 Class B） | PH-11〜PH-13 |
| A4. ドキュメント一貫性 | 設計→実装→テストのトレーサビリティ | 全フェーズ |
| A5. 数値正確性 | product_spec 数値との整合確認 | PH-04〜PH-10 |

**制約**: TCL3ツールの全出力は TORA による人間レビューが必須。自動承認は不可。
