# コーディングガイドライン
## 洗濯機BLDCモータ制御ECU

文書番号: WMC-CGL-001 / 版数: 2.0 / 作成日: 2026-03-02

**適用規格**: MISRA C:2012 + IEC 60730 Annex H
**対象言語**: C99 (ISO/IEC 9899:1999)
**対象MCU**: Renesas RL78/G14 (R5F104BG)
**SPICE**: SWE.2 WP 11-05

## 1. 適用範囲

### 1.1 対象コード

| 対象 | 説明 |
|------|------|
| 安全関連コード | SA-001〜SA-007（Class B モジュール） |
| 非安全コード | SA-008 COM（UART通信）— 本ガイドライン準拠を推奨 |
| 対象外 | テストコード、ビルドスクリプト、エクスポートスクリプト |

### 1.2 適用強度レベル

| レベル | 定義 | 該当SA |
|--------|------|--------|
| **A（最高）** | 安全状態遷移、異常検出、自己診断 | SA-002 (APP_SAF), SA-007 (DIAG) |
| **B（高）** | モータ制御、WDT管理、イベント管理 | SA-001 (APP_MOT), SA-004 (WDT_MGR), SA-005 (DEM), SA-006 (MEM_MGR) |
| **C（中）** | HW抽象化、通信 | SA-003 (HAL), SA-008 (COM) |

## 2. 言語規格・コンパイラ設定

### 2.1 言語規格

**C99** (ISO/IEC 9899:1999) — RL78/G14 CC-RL コンパイラ対応

### 2.2 コンパイラ設定

```makefile
# Renesas CC-RL コンパイラ推奨設定
CFLAGS = -cpu=S3 \
         -lang=c99 \
         -wall \
         -misra2012 \
         -optimize=2 \
         -goptimize
```

### 2.3 禁止言語機能

| 禁止機能 | 理由 | MISRA参照 |
|---------|------|----------|
| 動的メモリ (malloc/free/realloc) | ヒープ枯渇・断片化の予測不可 | Rule 21.3 |
| 再帰関数 | スタック消費量の静的解析不可 | Rule 17.2 |
| 浮動小数点演算 | 丸め誤差、RL78/G14にFPU非搭載 | — |
| VLA (可変長配列) | スタック消費量の予測不可 | Rule 18.8 |
| setjmp/longjmp | 制御フロー破壊 | Rule 21.4 |
| goto | 制御フロー複雑化 | Rule 15.1 |
| ポインタ算術 | 境界外アクセスリスク | Rule 18.4 |

## 3. MISRA C:2012 準拠方針

### 3.1 必須ルール (Mandatory/Required)

**全必須ルールを遵守**。逸脱が必要な場合は逸脱記録（Deviation Record）を作成。

主要ルール:

| ルール | 概要 | 適用レベル |
|--------|------|----------|
| Rule 1.3 | 未定義/未規定動作なし | A/B/C |
| Rule 8.4 | 互換型での関数宣言 | A/B/C |
| Rule 10.1-10.8 | 型変換の安全性 | A/B |
| Rule 11.1-11.9 | ポインタ変換の制限 | A/B |
| Rule 15.1 | goto禁止 | A/B/C |
| Rule 17.2 | 再帰禁止 | A/B/C |
| Rule 17.7 | 戻り値の使用 | A/B |
| Rule 21.3 | 動的メモリ禁止 | A/B/C |

### 3.2 推奨ルール (Advisory)

| ルール | 概要 | 備考 |
|--------|------|------|
| Rule 2.2 | デッドコードなし | 静的解析で検出 |
| Rule 15.5 | 関数は単一出口点 | レベルA/Bで必須化 |

### 3.3 静的解析ツール

```bash

# clang-tidy — コード品質チェック

# flawfinder — セキュリティ脆弱性チェック
flawfinder src/
```

## 4. 命名規則

### 4.1 関数命名

**形式**: `ModuleName_ActionVerb()` — PascalCase

```c
/* SA-001 APP_MOT */
void AppMot_Init(void);
void AppMot_Cyclic10ms(void);
void AppMot_EmergencyStop(void);

/* SA-002 APP_SAF */
void AppSaf_Init(void);
void AppSaf_Cyclic10ms(void);

/* SA-003 HAL */
uint16_t HalAdc_GetCurrent(void);
void HalPwm_SetDuty(uint8_t phase, uint16_t duty);
void HalPwm_AllStop(void);
bool HalGpio_GetLidState(void);

/* SA-007 DIAG */
DiagResult_t Diag_RunStartup(void);
void Diag_RunCyclic10ms(void);
```

### 4.2 変数命名

| 種別 | 命名規則 | 例 |
|------|---------|-----|
| グローバル変数 | `g_` プレフィックス + snake_case | `g_motor_rpm`, `g_diag_status` |
| 安全グローバル変数 | SafetyVar_t型 + `g_` | `SafetyVar_t g_motor_rpm` |
| static変数 | `s_` プレフィックス + snake_case | `s_loop_counter` |
| ローカル変数 | snake_case | `adc_value`, `is_overspeed` |
| パラメータ | snake_case | `target_rpm`, `error_code` |

### 4.3 定数・マクロ命名

**形式**: `UPPER_SNAKE_CASE`


## 5. 安全変数保護

### 5.1 逆コピー保護（SafetyVar_t）

全安全関連グローバル変数に適用。

```c
/* safety_types.h */
typedef struct {
    uint16_t value;     /* 実値 */
    uint16_t inverted;  /* ~value（ビット反転） */
} SafetyVar_t;

#define SAFETYVAR_SET(sv, val) do { \
    (sv).value = (uint16_t)(val);   \
    (sv).inverted = (uint16_t)(~(uint16_t)(val)); \
} while (0)

#define SAFETYVAR_GET(sv)   ((sv).value)

#define SAFETYVAR_CHECK(sv) \
    ((sv).value == (uint16_t)(~(sv).inverted))
```

### 5.2 クリティカルセクション

割込み/メインループ間の共有データ保護。

```c
/* RL78/G14 DI/EI マクロ */
#define ENTER_CRITICAL()  do { __DI()
#define EXIT_CRITICAL()   __EI(); } while (0)
```
