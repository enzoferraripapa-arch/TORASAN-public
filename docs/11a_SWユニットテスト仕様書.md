**適用規格**: IEC 60730 Class B / MISRA C:2012
**SPICE**: SWE.4 WP 08-50
**テストレベル**: ユニットテスト（ホワイトボックス）

---

## 1. テスト方針

### 1.1 テスト目的

PH-10で実装された20ユニット（UT-001〜UT-020）に対し、ホワイトボックステストを実施し、各ユニットが詳細設計仕様通りに動作することを検証する。

### 1.2 テスト戦略

- **テスト手法**: ホワイトボックステスト（境界値分析 + 同値分割法）
- **カバレッジ目標**:
  - 命令カバレッジ（C0）: ≧ 100%
  - 分岐カバレッジ（C1）: ≧ 90%
  - MC/DC: 安全関連関数（推奨）
- **SafetyVar_t検証**: 逆コピー保護の整合性チェック含む

### 1.3 テスト環境

- **コンパイラ**: GCC（ホストPC）
- **フレームワーク**: Unity + CMock
- **カバレッジ**: gcov / lcov
- **スタブ/モック**: HAL層はモック化、App層は直接テスト

### 1.4 スタブ/モック戦略（概要）

| 対象層 | 方針 | 詳細 |
|-------|------|------|
| HAL層（SA-003） | レジスタスタブ | MCUレジスタをRAM変数で模擬 |
| App層（SA-001, SA-002） | HALモック | CMockでHAL関数をモック化 |
| BSW層（SA-004〜008） | 個別モック | 依存モジュールをCMockで分離 |

※スタブ/モックの実装は別セッションで実施
