## 環境固有の知見

- **Windows cmd.exe vs Git Bash の PATH 差異** [S7]: cppcheck 等の外部ツールが cmd.exe で見つからない場合、Git Bash 経由のフォールバックが必要
- **Startup VBS とポート競合** [S8]: preview_start がポート3000を奪取 → ISS-008 で対応済み

## 開発プラクティス

- **Plan Mode を大規模変更に使う** [S3,S4]: 3ファイル以上・14スキル同時変更等は Plan Mode で設計方針を固めてから実装
- **Phase 実装と検証の分離** [S7]: 実装直後のレビューを習慣化。技術的負債の蓄積を防ぐ
- **検証ゲートの事前定義** [S2]: Plan 段階で G-01〜G-10 を定義 → 実装後に一括検証
- **ユーザーの意図は推測しない** [S3]: 分岐点では必ず確認（UX-1原則）

## 規格・ドメイン知見

- **プロセス手法の流用 ≠ 製品安全分類の適用** [S1]: V-model/FMEA は流用可能、ASIL 判定は車載専用
- **研究・調査系 PJ は軽量5フェーズ構成** [S9]: V-model 15フェーズは過剰
