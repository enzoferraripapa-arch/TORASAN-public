# ISO 26262 / IEC 60730 Dual Compliance Reference

## 1. Quick Reference: ASIL B Metrics

| Metric | Target | Formula |
|--------|--------|---------|
| SPFM (Single Point Fault Metric) | >= 90% | 1 - (lambda_SPF / lambda_total) |
| LFM (Latent Fault Metric) | >= 60% | 1 - (lambda_MPF_latent / lambda_total) |
| PMHF (Prob. Metric HW Failure) | < 1E-7/h | Sum of residual failure rates |

## 2. Diagnostic Coverage Categories

| DC Category | Coverage | Typical Mechanism |
|-------------|----------|-------------------|
| Low | 60% | Range/plausibility checks, voting 1oo2 |
| Medium | 90% | RAM March test, CRC on Flash, WDT |
| High | 99% | Dual-channel compare, ECC, redundant HW |

## 3. ISO 26262 Part 5 Safety Mechanism Catalog

| Table | Component | Class B Mechanisms |
|-------|-----------|-------------------|
| D.2 | CPU | Self-test (SW), lockstep (HW), instruction monitoring |
| D.3 | Program Counter | SW flow monitoring, WDT with time/logical monitoring |
| D.4 | Volatile Memory | March C-/Checkerboard, ECC, parity |
| D.5 | Non-Volatile Memory | CRC/checksum, dual storage, ECC |
| D.6 | I/O (digital) | Input compare, read-back, protocol checks |
| D.7 | I/O (analog) | Redundant ADC, range/plausibility, cross-check |
| D.8 | Bus | CRC, parity, dual bus |
| D.9 | Communication | CRC, sequence counter, timeout monitoring |
| D.10 | Clock | Independent clock monitor, crystal watchdog |
| D.11 | Supply Voltage | OV/UV monitor, BOD, redundant regulator |
| D.12 | Power Supply | Multi-rail monitoring, BIST |
| D.13 | Sensor | Dual sensor, plausibility/range |
| D.14 | Actuator | Current monitoring, position feedback |

## 4. IEC 60730 Class B vs ISO 26262 ASIL B Mapping

| IEC 60730 (Annex H) | ISO 26262 Equivalent | Notes |
|----------------------|---------------------|-------|
| H.2.16.5 CPU test | Part 5 Table D.2 | ISO requires more rigorous (BIST/lockstep) |
| H.2.19.6 RAM test | Part 5 Table D.4 | Both accept March C- |
| H.2.19.8 Flash test | Part 5 Table D.5 | CRC-32 satisfies both |
| H.2.18.10 Clock monitor | Part 5 Table D.10 | ISO may require independent oscillator |
| H.2.16.7 Interrupt test | Part 5 Table D.3 | ISO adds stack monitoring |
| H.2.18.11 Voltage monitor | Part 5 Table D.11 | ISO requires independent monitoring |
| H.2.14 WDT | Part 5 Table D.3 | Both: temporal + logical monitoring preferred |

## 5. Dual Compliance Checklist

- [ ] All IEC 60730 Annex H items covered (H.2.14-H.2.22)
- [ ] SPFM >= 90% demonstrated via FMEDA
- [ ] LFM >= 60% demonstrated via FMEDA
- [ ] PMHF < 1E-7/h calculated
- [ ] DC per safety mechanism documented with rationale
- [ ] Safety manual references both standards
- [ ] Self-test intervals justified (fault reaction time)
- [ ] Boot-time tests defined (startup diagnostics)
- [ ] Runtime tests defined (periodic diagnostics)
- [ ] Safe state defined and achievable within FTTI
- [ ] Hardware Architectural Metrics documented (Part 5 clause 8)
- [ ] Dependent failure analysis (DFA) performed

## 6. Common Pitfalls

| # | Pitfall | Mitigation |
|---|---------|------------|
| 1 | Claiming Class B compliance satisfies ASIL B | Class B is necessary but NOT sufficient; ASIL B requires HW metrics + SPICE |
| 2 | Using only periodic March test without justifying interval | Calculate FTTI; ensure test period < FTTI/2 |
| 3 | Missing clock cross-monitoring | Independent oscillator or external crystal WDT required |
| 4 | No common-cause failure analysis between channels | Perform DFA per Part 9; diversity is key |
| 5 | Assuming SW-only safety mechanisms achieve High DC | SW mechanisms typically cap at Medium (90%) DC |

## 7. Key Definitions

| Term | Definition |
|------|-----------|
| FTTI | Fault Tolerant Time Interval: time from fault to hazard |
| Safe State | System condition where no unreasonable risk exists |
| SPFM | Metric for single-point and residual faults |
| LFM | Metric for latent (multi-point) faults |
| PMHF | Probabilistic metric for random HW failures |

## 8. Safety Classification Scope Rules

### 8.1 ASIL判定は車載専用

ASIL判定（S×E×C → ASIL QM/A/B/C/D）は ISO 26262 Part 3 Table 4 に定義された車載専用の分類方法。
判定基準が道路交通の運用状況（走行速度・交通量・運転者の制御可能性）に依存するため、
車載以外の製品に ASIL判定を適用することは技術的に不適切。

### 8.2 各規格の分類方法

| 規格 | 分類方法 | 分類単位 | 決定手法 |
|------|---------|---------|---------|
| ISO 26262 | ASIL判定 | ASIL QM/A/B/C/D | S×E×C (Part 3 Table 4) |
| IEC 60730 | Class分類 | Class A/B/C | 機能種別 (§H.2) |
| IEC 61508 | SIL判定 | SIL 1/2/3/4 | リスクグラフ (Part 5) |
| IEC 62304 | SWクラス | Class A/B/C | 安全寄与度 (§4.3) |

### 8.3 プロセス手法流用 vs 製品分類の区別

- **流用可能**: V-model, SPICE, FMEA, FTA, FMEDA, FTTI分析, DC計算, S×E×C（補足リスク分析として）
- **流用不可**: ASIL判定結果を他製品の安全分類として使用
- **注意**: project.json の `standard.asil` は product_override 設定時「プロセス厳密度の指標」として機能し、製品の安全分類ではない

### 8.4 誤適用の検出ルール

```
IF product_override != null THEN
  ASIL判定は製品分類として非適用
  成果物中の "ASIL" は不整合の可能性あり
  → /select-standard check で検証推奨
END IF
```
