# Claude Code 運用リファレンス

## 1. コンテキストウィンドウ管理

### コンパクション（自動圧縮）

| 項目 | 内容 |
|------|------|
| トリガー | コンテキスト使用率 ~80%（約160,000トークン） |
| 動作 | 古い会話を要約し、重要なコード・決定・ファイルパスを保持 |
| 失われるもの | 初期メッセージの逐語的内容、中間推論、初期エラーメッセージ |
| 保持されるもの | CLAUDE.md（ディスクから再読込）、主要な決定・アーキテクチャ選択 |

### コンテキスト節約テクニック

| テクニック | 効果 | 方法 |
|-----------|------|------|
| `/clear` で区切る | 高 | 無関係なタスク間で実行、不要な文脈を排除 |
| サブエージェント委任 | 高 | 調査タスクを Task ツールで委任、結果サマリのみ受け取る |
| スコープを絞る | 中 | 「src/auth/ のトークン更新を調査」vs「コードベースを調査」 |
| Plan Mode 活用 | 中 | 3ファイル以上の変更前に設計、~40%トークン節約 |
| `/compact` 手動実行 | 中 | 70%使用時に「/compact API変更に集中」で焦点を指定 |

### セッション分割の判断基準

- 完全に無関係なタスクに切り替える時
- コンテキスト 60-70% 超過時
- 同じ問題で2回以上修正した時（文脈が汚染されている）
- 調査フェーズと実装フェーズの自然な境界

## 2. メモリシステム

### CLAUDE.md 読み込み階層

| 優先度 | スコープ | パス | タイミング |
|--------|---------|------|----------|
| 1（最高） | マネージド | OS固有のシステムパス | 常時、除外不可 |
| 2 | プロジェクト | `./CLAUDE.md` | 起動時（全文） |
| 3 | ユーザー | `~/.claude/CLAUDE.md` | 起動時（全文） |
| 4 | ローカル | `./CLAUDE.local.md` | 起動時（gitignore対象） |
| 5 | サブディレクトリ | `./subdir/CLAUDE.md` | オンデマンド |

**重要:** CLAUDE.md は 200行以内 を推奨。長いと遵守率が下がる。

### Auto Memory（自動メモリ）

| 項目 | 内容 |
|------|------|
| 保存先 | `~/.claude/projects/<project>/memory/` |
| 起動時読込 | `MEMORY.md` の先頭200行のみ |
| トピックファイル | オンデマンドで読み込み |
| 共有範囲 | 同一 git リポジトリ内の全ワークツリー・サブディレクトリ |
| 管理 | `/memory` コマンドで閲覧・編集・トグル |

### メモリの使い分け

| 保存先 | 用途 | 寿命 |
|--------|------|------|
| `.claude/knowledge/` | ドメイン知識（規格・技術リファレンス） | 永続（Git管理） |
| `~/.claude/projects/*/memory/` | プロジェクト固有の経験・決定 | 永続（ローカル） |
| session_state.md | 次セッション開始時の状態復元 | 1セッション |
| TodoWrite | セッション内タスク追跡 | セッション内のみ |

## 3. ワークツリー動作

### ライフサイクル

```
作成 → 作業 → セッション終了
  │              │
  │              ├─ 変更なし → 自動削除
  │              └─ 変更あり → ユーザーに確認
  │                            ├─ 保持 → ディレクトリ・ブランチ残存
  │                            └─ 削除 → 全て破棄
  │
  └─ ブランチ命名: claude/<adjective-noun>
     ディレクトリ: .claude/worktrees/<adjective-noun>/
```

### ワークツリー注意点

- `.claude/worktrees/` は `.gitignore` で除外済み
- 各ワークツリーで `npm install` が別途必要
- 同一ブランチを複数ワークツリーにチェックアウト不可
- サブエージェントも `isolation: worktree` でワークツリー分離可能

## 4. カスタムコマンド（スキル）

### ファイル構成

| 配置 | パス | スコープ |
|------|------|---------|
| プロジェクト | `.claude/commands/{name}.md` | このプロジェクト |
| ユーザー | `~/.claude/commands/{name}.md` | 全プロジェクト |

### $ARGUMENTS の動作

- `/skill-name arg1 arg2` → `$ARGUMENTS` = `"arg1 arg2"`
- 引数なし → `$ARGUMENTS` = `""`（空文字）
- スキル本文に `$ARGUMENTS` がない場合、末尾に自動追加

### スキル設計のベストプラクティス

| 原則 | 理由 |
|------|------|
| 500行以内 | 長すぎるとコンテキストを圧迫 |
| モード分岐 | 引数で動作を切り替え（"list" / "add" / "report"） |
| 検証ステップ含む | 実行後に結果を確認する手順を入れる |
| 関連スキル記載 | 次に使うべきスキルを提案 |
| ナレッジ参照 | `.claude/knowledge/` のリファレンスを参照 |

## 5. 設定管理

### settings.local.json 構造

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)"],
    "ask": ["Bash(git push *)"],
    "deny": ["Read(./.env)"]
  }
}
```

### 設定優先順位（高→低）

1. マネージド（MDM/システム）— 上書き不可
2. コマンドライン引数 — 一時的
3. ローカル（`.claude/settings.local.json`）— 個人、gitignore
4. プロジェクト（`.claude/settings.json`）— チーム共有
5. ユーザー（`~/.claude/settings.json`）— グローバル個人

### パーミッション評価順序

`deny → ask → allow`（最初にマッチしたルールが適用）

## 6. セッション引継ぎパターン

### セッション間で永続するもの / しないもの

| 永続する | 永続しない |
|---------|----------|
| CLAUDE.md（ディスクから再読込） | 会話メッセージ（`--continue` 除く） |
| Auto Memory | TodoWrite タスクリスト |
| Git コミット・ブランチ | ツール承認履歴 |
| `.claude/settings*.json` | 中間推論 |
| スキル・ナレッジファイル | サブエージェント結果（ファイル未保存分） |
| ワークツリー（保持選択時） | セッション内の修正・明確化 |

### 効果的な引継ぎ手順

1. **Git コミット = チェックポイント**: 意味のある単位でコミット
2. **ファイルに書く**: セッション末に発見・決定をファイルに記録
3. **`--continue` / `--resume`**: 直前セッションの継続
4. **PreCompact フック**: 圧縮前に自動で状態保存

### 引継ぎテンプレート

```markdown
# セッション引継ぎ — {date}
## 完了した作業
- {item1}
## 残タスク
- {item1}
## 重要な決定
- {decision1}: {理由}
## 変更したファイル
- {file1}: {変更内容}
## ブロッカー・未解決
- {issue1}
```

## 7. 凡ミス防止（必読）

詳細は `.claude/knowledge/error_prevention.md` を参照。

### 鉄則
- **パス slug は推測禁止** → `ls` で実在確認してから使う
- **Edit 前は自分で Read** → サブエージェントの Read は引き継がれない
- **3ファイル以上の変更はマルチエージェントレビュー必須**
- **3回失敗したら別アプローチ**（深入り禁止）

### セッション開始時の必須確認
1. `git status` で未コミット変更を把握
2. `ls ~/.claude/projects/ | grep {PJ名}` でメモリパス実在確認
3. `session_state.md` + `MEMORY.md` を読み込み
4. 前回の未解決改善項目を確認

## 8. パフォーマンスTips

| Tip | 効果 |
|-----|------|
| CLAUDE.md 200行以内 | 遵守率向上 + コンテキスト節約 |
| 2回修正失敗 → `/clear` + プロンプト書き直し | 汚染されたコンテキストをリセット |
| 調査はサブエージェントに委任 | メインコンテキストの保護 |
| `/compact` に焦点を指定 | 関連情報の保持率向上 |
| Git が最終的な永続化レイヤー | コミットは全てを生き残る |
