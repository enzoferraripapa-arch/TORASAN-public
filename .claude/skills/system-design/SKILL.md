# /system-design — システムアーキテクチャ設計

PH-06（システム設計）を支援するシステムアーキテクチャ設計の専門スキル。
ブロック図・インターフェース定義・安全機能配置・ASIL分解を行う。

引数: $ARGUMENTS（オプション。上記 argument-hint 参照。省略時は "all"）

> **数値参照ルール**: 本スキル内の製品名・IF 仕様・ASIL 値は project.json からの**例示値**です。
> 実行時は `project.json:description`（製品名）、`product_spec`（電気仕様）、`standard.asil`（ASIL）を Read して使用すること。

## Prerequisites（前提条件）

本スキル実行前に以下を確認:

| # | 条件 | 確認方法 | 未充足時 |
|---|------|---------|---------|
| 1 | project.json が存在する | `Read project.json` | `/session start` でプロジェクトを初期化 |

## システム構成要素

本プロジェクトのシステムは以下のサブシステムで構成（製品名・構成は `project.json:description` + docs/02_item_definition.md から取得）:

```
[洗濯機BLDCモータ制御システム]
├── SS-01: 電源サブシステム（AC→DC変換、電圧監視）
├── SS-02: モータ駆動サブシステム（インバータ、PWM制御）
├── SS-03: センササブシステム（電流・電圧・ホール・蓋検知）
├── SS-04: 制御サブシステム（RL78/G14 MCU + SW）
├── SS-05: 安全監視サブシステム（自己診断、WDT、安全遮断）
└── SS-06: ユーザーインターフェース（操作パネル、LED表示）
```

## 手順

### block モード — ブロック図生成
1. project.json の product_spec を読み込み
2. サブシステム間の接続関係を定義:

```
┌─────────────────────────────────────────────────┐
│              洗濯機BLDCモータ制御システム           │
│                                                   │
│  ┌────────┐    ┌────────┐    ┌────────┐          │
│  │ SS-01  │───▶│ SS-04  │───▶│ SS-02  │          │
│  │ 電源   │    │ 制御   │    │ モータ  │          │
│  │ 5V DC  │    │ MCU    │    │ 駆動    │          │
│  └────┬───┘    └───┬────┘    └────┬───┘          │
│       │            │  ▲           │              │
│       │            │  │           ▼              │
│       │       ┌────▼──┴──┐   ┌────────┐         │
│       │       │ SS-05    │   │ BLDC   │         │
│       │       │ 安全監視  │   │ モータ  │         │
│       │       └──────────┘   └────────┘         │
│       │            ▲                             │
│       │       ┌────┴─────┐   ┌────────┐         │
│       └──────▶│ SS-03    │   │ SS-06  │         │
│               │ センサ    │   │ UI     │         │
│               └──────────┘   └────────┘         │
└─────────────────────────────────────────────────┘
```

3. 各ブロックの入出力信号を一覧化
4. docs/06_system_design.md に出力

### interface モード — インターフェース定義
各サブシステム間の IF を定義:

| IF-ID | 接続元 | 接続先 | 信号種別 | 仕様 |
|-------|-------|-------|---------|------|
| IF-001 | SS-01 | SS-04 | 電源 | DC 5V ±5%, max 200mA |
| IF-002 | SS-04 | SS-02 | PWM | 3相PWM, 20kHz, 0-100% duty |
| IF-003 | SS-03→ADC | SS-04 | アナログ | 電流信号 0-5V (100Hz sampling) |
| IF-004 | SS-03→ADC | SS-04 | アナログ | 電圧信号 0-5V (10Hz sampling) |
| IF-005 | SS-03→Hall | SS-04 | デジタル | ホール信号 3ch, TTL |
| IF-006 | SS-03→Lid | SS-04 | デジタル | 蓋検知 GPIO, Active-Low |
| IF-007 | SS-04 | SS-05 | 内部 | 診断結果、安全状態遷移 |
| IF-008 | SS-05 | SS-02 | ハード遮断 | 安全遮断信号 (MCU非依存) |

各IFについて:
- 電気的仕様（電圧レベル、電流、インピーダンス）
- タイミング仕様（遅延、サンプリング周期）
- 異常時の振る舞い（断線、短絡、ノイズ）

### allocation モード — 安全機能配置
1. 安全目標(SG)を読み込み
2. 各SGをサブシステムに割当:

| SG | 安全機能 | 主担当 | 副担当 | 検出手段 |
|----|---------|-------|-------|---------|
| SG-001 | 過速度防止 | SS-04(SW) | SS-05(HW) | 速度監視 + 独立比較器 |
| SG-002 | 過電流防止 | SS-04(SW) | SS-02(HW) | ADC監視 + ヒューズ |
| SG-003 | 蓋開放保護 | SS-04(SW) | SS-05(HW) | GPIO + HWインターロック |
| SG-005 | MCU異常対応 | SS-05 | — | WDT + 自己診断 |
| SG-006 | 電圧異常対応 | SS-04(SW) | SS-01(HW) | ADC + UVLO |

3. 各安全機能の独立性・冗長性を評価

### asil モード — ASIL分解
ISO 26262 Part 9 に基づく ASIL 分解:
- 現在の ASIL: `{project.json:standard.asil}`（`{standard.product_override}` `{standard.product_override_class}` 相当）
- 分解の検討（ASIL B の場合の例）:
  - ASIL B = ASIL B(D) + QM（冗長なし）
  - ASIL B = ASIL A(D) + ASIL A(D)（HW+SW分解）
- 各サブシステムへの ASIL 割当を決定
- 独立性要件の検証

### review モード — 設計レビュー
以下の観点でレビュー:
1. **完全性**: 全SGがサブシステムに割り当てられているか
2. **整合性**: IF定義に矛盾がないか
3. **安全性**: 単一故障でSG違反が起きないか
4. **実現性**: RL78/G14 のリソースで実現可能か
5. **テスタビリティ**: 各IFが検証可能か

| 障害 | 検出方法 | 復旧手順 |
|------|---------|---------|
| 入力データ不足 | 必要ファイル/キーの不在 | 不足項目を報告し、前提スキルの実行を促す |
| 処理中断 | 予期しないエラー | エラー内容を報告。変更途中のファイルがあれば `git checkout` で復元 |

## 注意事項
- product_spec の数値を必ず参照（ハードコーディング禁止）
- HW/SW の責任分界を明確にする（PH-07/PH-08 への入力となる）
- 安全機能は独立性を確保（共通原因故障の排除）
- TSR への展開を見据えた粒度で設計する
