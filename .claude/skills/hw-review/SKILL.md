# /hw-review — ハードウェア設計レビューチェックリスト

PH-07（HW設計）および PH-12（HW統合テスト）に対応するハードウェア設計レビューを実施する。

引数: $ARGUMENTS（オプション。上記 argument-hint 参照。省略時は "all"）

## Prerequisites（前提条件）

本スキル実行前に以下を確認:

| # | 条件 | 確認方法 | 未充足時 |
|---|------|---------|---------|
| 1 | project.json が存在する | `Read project.json` | `/session start` でプロジェクトを初期化 |

## チェックリスト

### 1. 回路設計（circuit）

#### 電源回路
- [ ] 入力電圧範囲の確認（AC 100V±10% or 200V±10%）
- [ ] DC 5V レギュレータの出力精度（±5%以内）
- [ ] 突入電流制限回路の有無
- [ ] 過電圧保護（TVSダイオード等）
- [ ] フィルタコンデンサの容量・ESR
- [ ] 電圧モニタ回路（4.5V〜5.5V窓、product_spec.voltage_monitor）

#### インバータ回路
- [ ] MOSFET/IGBT の電圧・電流定格マージン（≧2倍）
- [ ] ゲートドライバのデッドタイム設定（貫通電流防止）
- [ ] フリーホイールダイオードの逆回復時間
- [ ] 電流検出用シャント抵抗（精度、温度特性、定格電力）
- [ ] 放熱設計（ヒートシンク、サーマルパッド）

#### MCU周辺
- [ ] クロック回路（32MHz 発振子、負荷容量）
- [ ] サブクロック（15kHz LOCO）の精度
- [ ] リセット回路（パワーオンリセット、外部リセット）
- [ ] WDT 外部回路（必要に応じて外部WDT併用）
- [ ] デカップリングコンデンサ（VDD/VSS 各ピン近傍）

### 2. 部品選定（component）

- [ ] 全部品の動作温度範囲（-40℃〜+85℃ or +105℃）
- [ ] ディレーティング基準:
  - 抵抗: 定格電力の 50% 以下
  - コンデンサ: 定格電圧の 80% 以下（電解）/ 50%以下（セラミック）
  - 半導体: Tj max の 80% 以下
- [ ] 部品の入手性・EOL リスク
- [ ] セカンドソースの有無
- [ ] RoHS / REACH 適合

### 3. EMC（emc）

- [ ] ESD 保護（外部I/O端子にTVSダイオード）
- [ ] EMI フィルタ（電源ライン、信号ライン）
- [ ] PWM ノイズ対策（スナバ回路、シールド）
- [ ] PCB パターン設計:
  - 電力ラインと信号ラインの分離
  - グランドプレーンの連続性
  - ADC 入力のガードリング
- [ ] 規格適合目標: CISPR 14-1 / IEC 61000-4-x

### 4. 安全回路（safety）

- [ ] 独立した安全遮断パス（MCU 故障時もモータ停止可能）
- [ ] 蓋インターロック回路（GPIO + ハードウェアインターロック）
- [ ] 過電流保護（ヒューズ + 電子式 OCP）
- [ ] 過速度検出の独立性（ソフト監視 + ハード比較器）
- [ ] 電源異常検出（ブラウンアウト検出、UVLO）
- [ ] 安全関連信号の冗長化（二重読取 or 多数決）
- [ ] IEC 60730 Class B 要求への適合:
  - 付録 H: 自己診断項目の HW サポート
  - 単一故障耐性の確認

### 5. 統合テスト項目（integration）— PH-12 対応

- [ ] ADC 精度測定（±1LSB @ 10bit = ±4.88mV）
- [ ] WDT タイムアウト実測（100ms ±10%）
- [ ] クロック周波数実測（32MHz ±4%）
- [ ] 電圧モニタ閾値実測（4.5V / 5.5V トリップポイント）
- [ ] モータ起動・停止シーケンス動作確認
- [ ] 緊急停止応答時間（<1ms）
- [ ] 蓋開放→モータ停止応答時間（<200ms）
- [ ] 電源瞬断耐性テスト
- [ ] 温度サイクルテスト（-20℃〜+60℃）
- [ ] 連続運転テスト（定格負荷 × 規定時間）

## Error Handling（エラー時の対応）

| 障害 | 検出方法 | 復旧手順 |
|------|---------|---------|
| 入力データ不足 | 必要ファイル/キーの不在 | 不足項目を報告し、前提スキルの実行を促す |
| 処理中断 | 予期しないエラー | エラー内容を報告。変更途中のファイルがあれば `git checkout` で復元 |

## 注意事項
- 回路図・基板レイアウトの実物がない場合は仕様ベースでレビュー
- 実測データがない項目は「未確認」として記録（推測で PASS にしない）
- レビュー結果は process_records/SYS.3_system_architecture.md に記録
- 不合格項目は problemLog（project.json）に登録を推奨
