# /systest-design — システムテスト設計

PH-13（システムテスト）/ SYS.5（システム適格性テスト）を支援するシステムレベルテスト専門スキル。
安全目標(SG)・機能安全要求(FSR)に対するシステムテスト仕様を設計する。

引数: $ARGUMENTS（オプション。上記 argument-hint 参照。省略時は "all"）

> **数値参照ルール**: テストシナリオ内の閾値（rpm、mA、ms、V 等）は project.json product_spec からの**例示値**です。
> 実行時は必ず `project.json` を Read して最新値を使用してください。

## 手順

### spec モード — システムテスト仕様生成
1. docs/03_hara.md から SG を読み込み
2. docs/04_fsc.md から FSR を読み込み
3. 各 SG/FSR に対してシステムテスト仕様を生成:

```
ST-{XXX}: {システムテスト名}
  対象要件: SG-{XXX} / FSR-{XXX}
  テスト目的: {SG/FSR の検証目的}
  テスト環境:
    - 実機 / HIL(Hardware-in-the-Loop) / シミュレーション
    - モータ負荷: {定格/無負荷/過負荷}
    - 電源条件: {定格/上限/下限}
    - 温度条件: {常温/高温/低温}
  前提条件: {セットアップ}
  手順:
    1. {step}
    2. {step}
  期待結果: {system-level expected behavior}
  判定基準: {定量的な合否基準}
  ASIL: {B}
```

### scenario モード — テストシナリオ設計
安全目標ごとの統合テストシナリオ:

#### SG-001: 過速度防止テスト
```
シナリオ: 定格運転中にモータ負荷急減→過速度発生
手順:
  1. 定格負荷(1200rpm)で安定運転を確認
  2. 負荷を急激に低減（モータ加速を誘発）
  3. 速度が1500rpmに到達する過程を計測
  4. 過速度検出 → 緊急停止を確認
判定:
  - 検出: 1500rpm到達から100ms以内
  - 停止: 検出から1ms以内にPWM=0
  - 復帰: 手動リセットまで再起動不可
```

#### SG-002: 過電流防止テスト
```
シナリオ: モータ拘束→過電流発生
手順:
  1. 低速運転中にモータ軸を拘束
  2. 電流上昇を計測
  3. 8A超過検出 → 電流遮断を確認
判定:
  - 検出: 8A超過から10ms以内
  - 遮断: 検出から1ms以内
```

#### SG-003: 蓋開放テスト
```
シナリオ: 運転中に蓋を開放
手順:
  1. 定格運転中に蓋を開放
  2. GPIO検知→モータ停止を確認
判定:
  - 検知から200ms以内にモータ停止
```

#### SG-005: MCU異常テスト
```
シナリオ群:
  a. WDTタイムアウト: メインループを意図的に停止→リセット確認
  b. RAM故障注入: テスト領域に不正値書込→March C検出確認
  c. ROM CRC不一致: 期待値を改変→CRC検出確認
  d. クロック異常: テストモードでクロック偏差注入→検出確認
```

#### SG-006: 電圧異常テスト
```
シナリオ:
  a. 電圧低下: 電源を4.5V以下に低下→安全停止確認
  b. 電圧上昇: 電源を5.5V以上に上昇→安全停止確認
  c. 瞬断: 電源を瞬時遮断(10ms〜100ms)→復帰/リセット確認
```

### acceptance モード — 受入基準定義
システムテスト全体の合格基準:

```
=== システムテスト受入基準 ===

[必須合格条件]
1. 全 SG に対する ST が PASS であること
2. 安全状態遷移の応答時間が FTTI 以内であること
3. 自己診断の DC が IEC 60730 Class B 要求を満たすこと
4. 連続運転テスト（{hours}時間）で異常なし

[環境条件]
- 温度: -20℃〜+60℃（動作保証範囲）
- 電源: 4.5V〜5.5V（仕様範囲内）
- 負荷: 0%〜100%（無負荷〜定格）

[合格率要求]
- 安全関連テスト: 100% PASS（例外なし）
- 非安全関連テスト: 95% PASS
```

## Error Handling（エラー時の対応）

| 障害 | 検出方法 | 復旧手順 |
|------|---------|---------|
| 入力データ不足 | 必要ファイル/キーの不在 | 不足項目を報告し、前提スキルの実行を促す |
| 処理中断 | 予期しないエラー | エラー内容を報告。変更途中のファイルがあれば `git checkout` で復元 |

## 注意事項
- システムテストは実機（または HIL）環境で実施前提
- シミュレーションのみで合格にできない項目を明記
- 故障注入テストは安全な環境で実施（モータ負荷制限）
- 数値基準は project.json product_spec から取得
